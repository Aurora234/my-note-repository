# 复杂度

算法复杂度用来**衡量算法运行效率和资源消耗**，主要分为两类：**时间复杂度**和**空间复杂度**，统一用 **大 O 表示法（Big O Notation）** 描述。

## 一、时间复杂度（Time Complexity）

表示**随着数据规模 n 增大，算法执行次数的增长趋势**，不关心具体耗时，只看增长快慢。

### 常见复杂度从快到慢排序

1. **O (1) 常数阶**

   执行次数固定，和数据规模无关。

   例：取数组第一个元素、简单赋值。

2. **O (log n) 对数阶**

   规模每次减半，增长极慢。

   例：二分查找。

3. **O (n) 线性阶**

   执行次数和数据量成正比。

   例：遍历一遍数组。

4. **O (n log n) 线性对数阶**

   高效排序算法的标准复杂度。

   例：快速排序、归并排序。

5. **O (n²) 平方阶**

   两层嵌套循环，数据变大后效率急剧下降。

   例：冒泡排序、选择排序。

6. **O (n³) 立方阶**

   三层嵌套循环，一般只用于小规模数据。

7. **O (2ⁿ) 指数阶**

   增长爆炸，n 稍大就无法运行。

   例：暴力枚举子集。

8. **O (n!) 阶乘阶**

   最慢，仅用于极小规模。

   例：全排列暴力枚举。

------

## 二、空间复杂度（Space Complexity）

表示算法**占用内存随数据规模 n 的增长趋势**。

- **O(1)**：原地操作，额外空间固定
- **O(n)**：需要开辟和数据量等大的额外空间
- **O(n²)**：二维数组等

------

## 三、大 O 表示法的核心规则

1. 只保留**增长最快的项**

   O (n² + 3n + 5) → 简化为 O(n²)

2. 忽略**常数系数**

   O(3n) → O(n)

3. 只看**最坏情况**

   比如查找，默认按找不到、遍历全部来算

## 简单排序算法

### 选择排序

1. 从0下标开始，向后看找最小的数的下标，

2. 交换前方有序数列的最后一位的数字。
3. 循环n次

```java
	public static void selectionSort(int[] arr) {
		if (arr == null || arr.length < 2) {
			return;
		}
		for (int i = 0; i < arr.length - 1; i++) {
			int minIndex = i;
			for (int j = i + 1; j < arr.length; j++) {
				minIndex = arr[j] < arr[minIndex] ? j : minIndex;
			}
			swap(arr, i, minIndex);
		}
	}

	public static void swap(int[] arr, int i, int j) {
		int tmp = arr[i];
		arr[i] = arr[j];
		arr[j] = tmp;
	}
```



### 冒泡排序

1. 从头开始，比较相邻两个数
2. 交换位置
3. 直到最后，最大（小）的数在最后
4. 循环n次

```java
public static void bubbleSort(int[] arr) {
    if (arr == null || arr.length < 2) {
       return;
    }
    for (int e = arr.length - 1; e > 0; e--) {
       for (int i = 0; i < e; i++) {
          if (arr[i] > arr[i + 1]) {
             swap(arr, i, i + 1);
          }
       }
    }
}

public static void swap(int[] arr, int i, int j) {
    arr[i] = arr[i] ^ arr[j];
    arr[j] = arr[i] ^ arr[j];
    arr[i] = arr[i] ^ arr[j];
}
```

## 异或差异

#### 注：异或运算

- **归零律 (自反性)**
  任何数与自身进行异或运算，结果都为 0。
  `a ^ a = 0`
- **恒等律**
  任何数与 0 进行异或运算，结果都为其本身。
  `a ^ 0 = a`
- **交换律**
  异或运算的顺序不影响最终结果。
  `a ^ b = b ^ a`
- **结合律**
  多个数进行异或运算时，运算的分组方式不影响结果。
  `(a ^ b) ^ c = a ^ (b ^ c)`
- **还原律 (可逆性)**
  一个数连续异或同一个数两次，结果会还原为原数。这个性质可以从 `a ^ a = 0` 和 `a ^ 0 = a` 推导出来。`a ^ b ^ b = a ^ (b ^ b) = a ^ 0 = a`

#### **不使用临时变量交换两个数**

- **原理**：利用**还原律**，通过三步操作将两个变量的值“揉”在一起再“拆”开。
- **示例**：交换变量 `a` 和 `b` 的值。
  1. `a = a ^ b` (此时 `a` 存储的是两数异或的结果)
  2. `b = a ^ b` (相当于 `(a ^ b) ^ b`，根据还原律，结果为原 `a` 的值，并赋给 `b`)
  3. `a = a ^ b` (此时 `a` 仍是 `a ^ b`，而 `b` 已是原 `a` 的值。所以这步相当于 `(a ^ b) ^ a`，根据交换律和还原律，结果为原 `b` 的值，并赋给 `a`)
     经过这三步，`a` 和 `b` 的值就成功交换了。

#### **查找数组中“只出现一次的数字”**

如果一个数组中，除了一个元素只出现一次，其余所有元素都出现两次，那么将数组中所有元素进行异或运算，最终结果就是那个只出现一次的数字。

- **原理**：根据**交换律**和**结合律**，我们可以将相同的数字放在一起运算。再根据**归零律** (`a ^ a = 0`)，所有出现两次的数字异或后都变为0。最后，根据**恒等律** (`0 ^ a = a`)，0与那个唯一的数字异或，结果就是它本身。
- **示例**：对于数组 `[1, 2, 1, 3, 2]`，计算 `1 ^ 2 ^ 1 ^ 3 ^ 2`。
  - 重新分组：`(1 ^ 1) ^ (2 ^ 2) ^ 3`
  - 应用归零律：`0 ^ 0 ^ 3`
  - 应用恒等律：`3`
  - 最终结果为 **3**。

---

#### **数组中只出现一次的两个数字**

**第一步：整体异或，得到两个目标数的“差异值”**

首先，将数组中所有数字进行异或运算。根据异或的性质（`a ^ a = 0`），所有出现两次的数字都会相互抵消变为0。最终，整个数组的异或结果 `xor_all` 就等于那两个只出现一次的数字（我们称之为 `a` 和 `b`）的异或值。

```
xor_all = a ^ b
```

因为 `a` 和 `b` 是两个不同的数字，所以 `xor_all` 的结果一定不为0。这意味着在 `xor_all` 的二进制表示中，至少有一位是 `1`。

**第二步：找到“差异位”，作为分组依据**

`xor_all` 中为 `1` 的位，恰恰就是 `a` 和 `b` 在该位上不同的地方（一个为0，一个为1）。我们可以利用这个“差异位”作为标准，将原数组中的所有数字分成两组。

通常，为了方便，我们会选择 `xor_all` 二进制表示中**最低位的那个 `1`**。有一个非常巧妙的位运算技巧可以快速获取这个值：

```
diff = xor_all & (-xor_all)
```

这个 `diff` 的值只有一个二进制位是 `1`，这个 `1` 就是我们用来分组的“差异位”。

**第三步：分组异或，分别求出两个数字**

现在，我们根据每个数字与 `diff` 进行按位与（`&`）运算的结果来分组：

- **第一组**：数字 `num` 满足 `(num & diff) == 0`。这意味着该数字在“差异位”上是 `0`。
- **第二组**：数字 `num` 满足 `(num & diff) != 0`。这意味着该数字在“差异位”上是 `1`。

这样分组后，会产生两个关键效果：

1. **目标数字被分离**：因为 `a` 和 `b` 在“差异位”上必然不同，所以它们会被分到不同的组里。
2. **成对数字不分离**：所有出现两次的数字，它们的值完全相同，所以在“差异位”上也一定相同，它们会一起被分到同一个组里。

最后，我们分别对这两个组内的所有数字进行异或运算。在每个组内部，成对的数字会再次相互抵消，最终每个组剩下的结果就是我们要求的 `a` 和 `b`。

```java
	public static void printOddTimesNum2(int[] arr) {
		int eO = 0, eOhasOne = 0;
		for (int curNum : arr) {
			eO ^= curNum;
		}
		int rightOne = eO & (~eO + 1);
		for (int cur : arr) {
			if ((cur & rightOne) != 0) {
				eOhasOne ^= cur;
			}
		}
		System.out.println(eOhasOne + " " + (eO ^ eOhasOne));
	}
```





## 二分

**二分查找不仅仅用于有序数组，只要你能通过中间状态判断目标值在左侧还是右侧，就可以使用二分。**

### 基本查找

```java
	public static boolean exist(int[] sortedArr, int num) {
		if (sortedArr == null || sortedArr.length == 0) {
			return false;
		}
		int L = 0;
		int R = sortedArr.length - 1;
		int mid = 0;
		while (L < R) {
			mid = L + ((R - L) >> 1);
			if (sortedArr[mid] == num) {
				return true;
			} else if (sortedArr[mid] > num) {
				R = mid - 1;
			} else {
				L = mid + 1;
			}
		}
		return sortedArr[L] == num;
	}
```

### 在arr上，找满足>=value的最左位置

```java
	public static int nearestIndex(int[] arr, int value) {
		int L = 0;
		int R = arr.length - 1;
		int index = -1;
		while (L < R) {
            // 知道L和R范围内没有数字为止，无法提前终止
			int mid = L + ((R - L) >> 1);
			if (arr[mid] >= value) {
				index = mid;
				R = mid - 1;
			} else {
				L = mid + 1;
			}
		}
		return index;
	}
```

### 局部最小

arr中，无序，相邻数字不相等，求一个局部最小（某位置数字小于两边，若在边界只考虑一边）即可，要求O(n)

```java
        public static int getLocalMinIndex(int[] arr) {
            if (arr == null || arr.length == 0) {
                return -1;
            }
            int n = arr.length;
            if (n == 1) {
                return 0;
            }
            if (arr[0] < arr[1]) {
                return 0;
            }
            if (arr[n - 1] < arr[n - 2]) {
                return n - 1;
            }
            int left = 1;
            int right = n - 2;
            while (left <= right) {
                int mid = left + ((right - left) >> 1); // 防止溢出的写法，等同于 (left + right) / 2
                if (arr[mid] < arr[mid - 1] && arr[mid] < arr[mid + 1]) {
                    return mid;
                }
                else if (arr[mid] > arr[mid - 1]) {
                    right = mid - 1;
                }
                else {
                    left = mid + 1;
                }
            }
            return -1;
        }
```

```java
        public static int getMax(int[] arr){
            return process(arr, 0, arr.length-1);
        }
        public static int process(int[] arr, int left, int right){
            if(left == right){
                return arr[left];
            }
            int mid = left + ((right - left) >> 1);
            int leftMax =process(arr, left, mid);
            int rightMax = process(arr, mid + 1, right);
            return Math.max(leftMax, rightMax);
        }
```

#### master公式

Master公式适用于符合以下标准形式的递归关系式：

```
T(n) = a * T(n/b) + O(nᵈ)
```

其中：

- **`T(n)`**: 表示处理规模为 `n` 的问题所需的时间复杂度。
- **`a`**: 表示每次递归调用产生的**子问题数量**。
- **`n/b`**: 表示每个子问题的**规模**（假设所有子问题规模相同）。
- **`O(nᵈ)`**: 表示除了递归调用本身之外，进行**问题分解和结果合并**等操作所需的时间复杂度。

**重要前提**：Master公式要求每次递归产生的子问题规模必须是一致的（即都是 `n/b`），否则无法直接应用。

通过比较 `log_b(a)`（以 `b` 为底 `a` 的对数）和 `d` 的大小关系，可以直接得出时间复杂度：

**三种情况**

| 情况  | 条件           | 时间复杂度          |
| :---- | :------------- | :------------------ |
| **1** | `log_b(a) > d` | **O(n^(log_b(a)))** |
| **2** | `log_b(a) = d` | **O(nᵈ \* log n)**  |
| **3** | `log_b(a) < d` | **O(nᵈ)**           |

简单来说，时间复杂度由 `n^(log_b(a))` 和 `nᵈ` 中增长更快的那一项决定。如果两者增长速度相同，则需要额外乘以一个 `log n` 因子。

### 归并排序

默认左边和右边都有序，合并两个有序数列

```java
	public static void mergeSort(int[] arr) {
		if (arr == null || arr.length < 2) {
			return;
		}
		mergeSort(arr, 0, arr.length - 1);
	}

	public static void mergeSort(int[] arr, int l, int r) {
		if (l == r) {
			return;
		}
		int mid = l + ((r - l) >> 1);
		mergeSort(arr, l, mid);
		mergeSort(arr, mid + 1, r);
		merge(arr, l, mid, r);
	}

	public static void merge(int[] arr, int l, int m, int r) {
		int[] help = new int[r - l + 1];
		int i = 0;
		int p1 = l;
		int p2 = m + 1;
		while (p1 <= m && p2 <= r) {
			help[i++] = arr[p1] < arr[p2] ? arr[p1++] : arr[p2++];
		}
		while (p1 <= m) {
			help[i++] = arr[p1++];
		}
		while (p2 <= r) {
			help[i++] = arr[p2++];
		}
		for (i = 0; i < help.length; i++) {
			arr[l + i] = help[i];
		}
	}

```

#### 小和问题与逆序对问题

**小和问题**：

> 在一个数组中，每一个数左边比当前数小的数累加起来，叫做这个数组的小和。求一个数组的小和。例如：对于数组[1,3,4,2,5]
> 1左边比1小的数，没有；
> 3左边比3小的数，1；
> 4左边比4小的数，1、3；
> 2左边比2小的数，1；
> 5左边比5小的数，1、3、4、2；
> 所以小和为1+1+3+1+1+3+4+2=16

**逆序对问题**：

> 逆序对是指在一个序列中，满足 **i < j 且 a[i] > a[j]** 的有序对数量。

注意：merge时左右两边数组元素进行比较时，必须先拷贝右边

```java
	public static int smallSum(int[] arr) {
		if (arr == null || arr.length < 2) {
			return 0;
		}
		return mergeSort(arr, 0, arr.length - 1);
	}

	public static int mergeSort(int[] arr, int l, int r) {
		if (l == r) {
			return 0;
		}
		int mid = l + ((r - l) >> 1);
		return mergeSort(arr, l, mid) 
				+ mergeSort(arr, mid + 1, r) 
				+ merge(arr, l, mid, r);
	}

	public static int merge(int[] arr, int l, int m, int r) {
		int[] help = new int[r - l + 1];
		int i = 0;
		int p1 = l;
		int p2 = m + 1;
		int res = 0;
		while (p1 <= m && p2 <= r) {
			res += arr[p1] < arr[p2] ? (r - p2 + 1) * arr[p1] : 0;
			help[i++] = arr[p1] < arr[p2] ? arr[p1++] : arr[p2++];
		}
		while (p1 <= m) {
			help[i++] = arr[p1++];
		}
		while (p2 <= r) {
			help[i++] = arr[p2++];
		}
		for (i = 0; i < help.length; i++) {
			arr[l + i] = help[i];
		}
		return res;
	}
```

### 荷兰国旗问题

> 给定数组arr，将[l, r]范围内的数（当然默认是 [ 0 - arr.length - 1 ]），小于arr[r]（这里直接取数组最右边的值进行比较）放到数组左边，等于arr[r]放到数组中间，大于arr[r]放到数组右边。最后返回等于arr[r]的左, 右下标值。

```java
	public static int[] partition(int[] arr, int l, int r, int p) {
		int less = l - 1;
		int more = r + 1;
		while (l < more) {
			if (arr[l] < p) {
				swap(arr, ++less, l++);
			} else if (arr[l] > p) {
				swap(arr, --more, l);
			} else {
				l++;
			}
		}
		return new int[] { less + 1, more - 1 };
	}
```



### 快速排序

```java
	public static void quickSort(int[] arr) {

	public static void quickSort(int[] arr, int l, int r) {
		if (l < r) {
			swap(arr, l + (int) (Math.random() * (r - l + 1)), r);
			int[] p = partition(arr, l, r);
			quickSort(arr, l, p[0] - 1);
			quickSort(arr, p[1] + 1, r);

	public static int[] partition(int[] arr, int l, int r) {
		int less = l - 1;
		int more = r;
		while (l < more) {
			if (arr[l] < arr[r]) {
				swap(arr, ++less, l++);
			} else if (arr[l] > arr[r]) {
				swap(arr, --more, l);
			} else {
				l++;
			}
		}
		swap(arr, more, r);
		return new int[] { less + 1, more };
	}
```
