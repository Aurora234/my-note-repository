# 情感分析



## 基于同义词词典

**于同义词词典的方法**是情感分析中一种**基于规则**的经典方法。它通过“查字典”和“计分”的方式工作，具有简单、快速、无需训练的优点，但其准确性严重依赖于词典的质量和规则设计的完备性，在处理复杂语言时显得力不从心。在现代NLP中，它通常作为基线系统或辅助特征，与更先进的机器学习/深度学习方法结合使用。

### 核心思想

这种方法的核心思想非常简单直接：**通过查询一个预先构建好的、包含了词语情感极性的词典，来判断一段文本的情感倾向。**

它本质上是一种 **“查字典”** 的方法。这个字典不仅包含词语的同义词，更重要的是，它给每个词语打上了**情感标签**（例如，正面、负面、中性）或**情感分数**（例如，+3 表示非常正面，-2 表示比较负面）。

### 方法的工作流程

假设我们要判断句子 **“这部电影真棒，演员演技精湛，剧情却很枯燥。”** 的情感。

**第一步：构建情感词典（前提条件）**

我们需要一个情感词典。一个简单的情感词典可能长这样：

| 词语 | 情感极性 | 强度分数 |
| :--- | :------- | :------- |
| 棒   | 正面     | +3       |
| 精湛 | 正面     | +2       |
| 优秀 | 正面     | +2       |
| 枯燥 | 负面     | -2       |
| 糟糕 | 负面     | -3       |
| 美丽 | 正面     | +2       |
| 丑陋 | 负面     | -2       |
| ...  | ...      | ...      |

**第二步：文本预处理**

1. **分词：** 将句子分割成独立的词语。
   - `这 / 部 / 电影 / 真 / 棒 / ， / 演员 / 演技 / 精湛 / ， / 剧情 / 却 / 很 / 枯燥 / 。`
   - （注意：这里“真”和“很”是程度副词，需要特殊处理）

**第三步：查询词典并计算情感值**

1. 遍历每个分词后的词语，去情感词典里查找。
2. 找到匹配的词语，记录其情感分数：
   - `棒` -> +3
   - `精湛` -> +2
   - `枯燥` -> -2
   - 其他词语（如“电影”、“演员”、“剧情”）在词典中未找到，视为中性词，分数为0。
3. **（可选）处理程度副词和否定词：** 一个成熟的系统会考虑这些修饰词。
   - `真` 棒： “真”是加强语气的词，可以将“棒”的分数乘以一个系数（比如1.5）。所以 `棒` 的分数变为 +3 * 1.5 = +4.5。
   - `很` 枯燥：“很”也是加强词，`枯燥` 的分数变为 -2 * 1.2 = -2.4。
   - `却不枯燥`：如果遇到“不”这样的否定词，可以将其后词语的情感分数反转。`不枯燥` -> -(-2) = +2。

**第四步：汇总得分并得出结论**

- 将所有词语的情感分数相加：
  - 原始分数：+3 (棒) + +2 (精湛) + -2 (枯燥) = **+3**
  - 考虑程度副词后：+4.5 (真棒) + +2 (精湛) + -2.4 (很枯燥) = **+4.1**
- **制定判断规则：**
  - 如果总分 > 0，则判断为**正面**情感。
  - 如果总分 < 0，则判断为**负面**情感。
  - 如果总分 = 0，则判断为**中性**。

根据这个规则，我们的例子虽然提到了缺点（枯燥），但优点更突出，所以整体被判断为**正面评价**。

### 关键组成部分

一个完整的基于词典的系统通常包含以下几个词典：

1. **基础情感词库：** 核心词典，包含大量词语的情感极性和强度。
   - *例子：* `好 (+2)`, `坏 (-2)`, `美丽 (+2)`, `丑陋 (-2)`。
2. **否定词词典：** 包含可以反转情感倾向的词。
   - *例子：* `不`， `没`， `无`， `莫`。
3. **程度副词词典：** 包含可以加强或减弱情感强度的词，并为每个副词赋予一个权重。
   - *极强：* `极其`， `完美` (权重 ~2.0)
   - *强：* `非常`， `十分` (权重 ~1.5)
   - *一般：* `很`， `挺` (权重 ~1.2)
   - *弱：* `有点`， `稍微` (权重 ~0.5)
4. **连词词典：** 用于处理复杂句子结构，但简单实现中常被忽略。

## 基于计数的方法

### 核心思想

基于计数的方法的核心思想非常直观：**通过统计大规模文本数据中词语共同出现的频率，来量化它们之间的语义或语法关系。**

它的基本假设是 **“分布假说”**：**出现在相似上下文中的词语，具有相似的含义。** 例如，“咖啡”和“茶”经常出现在类似的语境中（如“喝”、“一杯”、“提神”），因此我们可以认为它们在语义上是相关的。

基于计数的方法旨在将文本中的词语转换为**数值向量**（即词向量），而这些向量的生成完全依赖于词语在语料库中的共现统计信息。

------

### 主要方法与工作流程

最经典的基于计数的方法是**词袋模型**及其扩展**TF-IDF**，以及用于生成稠密词向量的**共现矩阵与降维**技术。

#### 1. 词袋模型

这是最基础、最直观的基于计数的方法。

**工作流程：**

1. **构建词汇表：** 收集整个语料库（所有文档的集合）中所有不同的词语，形成一个词汇表。假设词汇表大小为 V。
2. **统计频率：** 对于每一篇文档，我们统计每个词语在文档中出现的次数。
3. **向量化：** 将每篇文档表示成一个长度为 V 的向量。向量的每一个位置对应词汇表中的一个词，其值就是该词在这篇文档中出现的次数。

**举例说明：**
假设我们的词汇表来自两个句子：

- 文档1: “我 喜欢 看 电影”
- 文档2: “我 喜欢 读 书”

**词汇表：** `[“我”， “喜欢”， “看”， “电影”， “读”， “书”]` (V=6)

| 词语 | 索引 |
| :--- | :--- |
| 我   | 0    |
| 喜欢 | 1    |
| 看   | 2    |
| 电影 | 3    |
| 读   | 4    |
| 书   | 5    |

现在，我们将两篇文档转换为向量：

- 文档1向量： `[1, 1, 1, 1, 0, 0]`
  - （“我”出现1次，“喜欢”1次，“看”1次，“电影”1次，“读”0次，“书”0次）
- 文档2向量： `[1, 1, 0, 0, 1, 1]`

**在电影评论分类中的应用：**

- 我们可以将每条电影评论看作一篇文档，用词袋模型将其转换为一个数值向量。
- 然后，将这些向量和它们对应的情感标签（正面/负面）输入到一个机器学习分类器（如朴素贝叶斯、逻辑回归、SVM）中进行训练。
- 分类器会学习到哪些词语的组合（即向量的模式）更倾向于正面评价，哪些更倾向于负面评价。

**缺点：**

- **高维稀疏：** 向量维度等于词汇表大小，可能高达数万甚至数百万，但每个文档向量中只有少量非零值。
- **忽略词序：** “电影不好看”和“不好看电影”的向量表示是完全一样的，但语义可能不同。
- **忽略语义：** 无法捕捉“电影”和“影片”之间的语义相似性，它们被视为两个完全独立的特征。

#### 2. TF-IDF

TF-IDF是词袋模型的增强版，它不仅考虑词频，还考虑词语在整个语料库中的重要性。

- **TF：** 词频，一个词在文档中出现的次数。TF越高，表示该词对当前文档越重要。
- **IDF：** 逆文档频率，一个衡量词语普遍重要性的指标。计算公式通常是 `log(总文档数 / 包含该词的文档数)`。如果一个词在太多文档中都出现（如“的”、“是”），其IDF值会很低，表明它区分文档的能力差。

**TF-IDF = TF × IDF**

**作用：** TF-IDF通过降低常见词的权重，提高稀有且重要词的权重，从而得到更好的文档向量表示。在电影评论中，“精彩绝伦”的TF-IDF值通常会远高于“电影”这个词。

#### 3. 词语级共现矩阵与词向量

这种方法旨在为**单个词语**生成有意义的向量表示，从而捕捉词语之间的语义关系。

**工作流程：**

1. **定义上下文窗口：** 选择一个固定大小的窗口（例如，窗口大小为1，即只看左右相邻的一个词）。
2. **构建共现矩阵：** 创建一个 |V| x |V| 的矩阵 X，其中 `X[i][j]` 表示在整个语料库中，词语 `j` 在词语 `i` 的上下文窗口中出现的次数。

**举例说明：**
语料库包含三个句子：

- “我 喜欢 电影”
- “我 喜欢 音乐”
- “我 享受 电影”

假设词汇表 V = `[我， 喜欢， 电影， 音乐， 享受]`，窗口大小为1。

我们构建共现矩阵（以“我”为例）：

- “我”的右边是“喜欢”和“享受”。
- 所以，在“我”的行中，“喜欢”计数+2，“享受”计数+1。

最终矩阵可能如下（仅示意）：

|          | 我   | 喜欢 | 电影 | 音乐 | 享受 |
| :------- | :--- | :--- | :--- | :--- | :--- |
| **我**   | 0    | 2    | 0    | 0    | 1    |
| **喜欢** | 2    | 0    | 1    | 1    | 0    |
| **电影** | 0    | 1    | 0    | 0    | 1    |
| **音乐** | 0    | 1    | 0    | 0    | 0    |
| **享受** | 1    | 0    | 1    | 0    | 0    |

**问题与优化：**

- 这个矩阵同样**高维且稀疏**。
- **解决方案：降维。** 使用诸如**主成分分析（PCA）** 或**奇异值分解（SVD）** 等技术，将这个巨大的稀疏矩阵压缩成一个低维的稠密矩阵（比如 |V| x 100）。这个新的、更小的矩阵的每一行，就是一个词语的**稠密词向量**。
- 在这些低维向量空间中，语义相似的词语（如“电影”和“影片”）其向量在空间中的位置会非常接近。

**GloVe模型** 就是基于这种思想的一个著名且成功的词向量模型，它本质上是对共现计数矩阵进行加权分解。

## 基于推理的方法

简单来说：

> **基于推理的方法**是指系统通过操作内部的知识表示，运用逻辑规则或数学机制，从已知信息中推导出新结论或理解深层含义的过程。它不仅仅是识别模式，更是进行“思考”。

为了帮助理解，我们与之前讨论的方法进行对比：

| 方法类型                | 核心思想                     | 例子（电影评论分类）                                         | 局限性                                 |
| :---------------------- | :--------------------------- | :----------------------------------------------------------- | :------------------------------------- |
| **基于词典的方法**      | 查字典，计分。               | 统计“优秀”、“烂片”等情感词的分数总和。                       | 无法处理反讽、逻辑关系和复杂上下文。   |
| **基于计数/统计的方法** | 统计词语共现模式。           | 机器学习模型学习到“剧情紧凑”和正面标签共现概率高。           | 仍然是浅层的相关性学习，缺乏深层理解。 |
| **基于推理的方法**      | **理解逻辑关系，进行推导。** | 理解到“如果导演是A，而这部电影是A拍的，那么它可能具有深度”这样的逻辑链。 | 实现难度大，需要大量结构化知识。       |

------

### 基于推理方法的核心组成部分

一个基于推理的系统通常包含以下几个要素：

1. **知识表示：**
   - 系统需要将世界知识、常识和特定领域知识以一种结构化的形式存储起来。
   - **形式可以是：** 知识图谱（例如，“张艺谋 -导演-> 《活着》”）、逻辑命题（例如，`∀x, 电影(x) ∧ 有深度(x) → 值得观看(x)`）、规则等。
2. **推理引擎：**
   - 这是系统的“大脑”，它操作知识表示，执行推理过程。
   - **推理方式包括：**
     - **演绎推理：** 从一般规则和前提推导出特定结论。如果前提为真，则结论必然为真。
       - **规则：** 所有由诺兰导演的电影都值得观看。
       - **事实：** 《信条》是由诺兰导演的。
       - **结论：** 《信条》是值得观看的。
     - **归纳推理：** 从多个特定事例中总结出一般规则。
       - **观察：** 看到100条评论说“这部电影剧情拖沓”并打了低分。
       - **归纳：** “剧情拖沓”可能与负面评价相关。
     - **溯因推理：** 为观察到的现象寻找最有可能的解释。
       - **观察：** 一条评论说：“这部电影完美地让我睡着了。”
       - **推理：** 这句话表面是“完美”，但结合常识（看电影不想睡着），最合理的解释是**反讽**，因此情感是负面的。

------

### 在电影评论分类中的具体应用

对于“这部电影的演员阵容豪华，但剧本一塌糊涂”这条评论：

- **基于词典的方法：** 看到“豪华”(+3) 和 “一塌糊涂”(-3)，分数相加为0，可能误判为“中性”。
- **基于推理的方法：**
  1. **理解逻辑结构：** 识别出“但”这个转折词，知道其后的内容通常更重要，表达了作者的真实意图。
  2. **常识推理：** 知道“剧本”是电影的核心要素，而“演员阵容”是次要要素。一个核心要素的严重缺陷（一塌糊涂）足以否定一个次要要素的优点（豪华）。
  3. **得出结论：** 尽管有优点，但核心缺点更致命，因此整体评价是**负面**的。

**另一个例子：处理反讽**
评论：“真是部‘杰作’啊，看得我每分钟都在看表。”

- **基于词典的方法：** 识别出“杰作”(+3)，可能误判为正面。
- **基于推理的方法：**
  1. **检测不一致性：** “杰作”是高度赞扬，但“每分钟都在看表”是极度无聊的表现。
  2. **常识与溯因推理：** 根据常识，一个人不会喜欢一部让他不停看表的电影。这种矛盾最合理的解释是，说话者在使用反讽。
  3. **得出结论：** “杰作”在此处是反话，真实情感是**负面**的。

### 实现技术：从传统到现代

#### 1. 传统符号主义方法

- **技术：** 使用一阶逻辑、描述逻辑、产生式规则系统。
- **流程：** 人工构建庞大的知识库（如Cyc）和推理规则。系统通过逻辑演算进行推理。
- **缺点：** 知识获取瓶颈（需要人工构建，成本高），难以处理不确定性和模糊性。

#### 2. 现代统计推理方法

这类方法将推理与概率结合起来，处理不确定性。

- **概率图模型：** 如贝叶斯网络。可以表示变量间的依赖关系，并进行概率推理。
  - *例子：* 构建一个网络，节点包括“导演水平”、“剧本质量”、“演员演技”、“最终评分”。已知“剧本质量”差，可以推断“最终评分”低的概率会增高。
- **基于知识图谱的推理：**
  - 系统从一个实体出发，沿着关系路径进行推理。
  - *例子：* 用户查询“推荐一部像《阿甘正传》一样温暖的电影”。
    - 系统在知识图谱中找到《阿甘正传》的标签是“温暖”、“励志”。
    - 然后推理：找到同样具有“温暖”和“励志”标签的电影，比如《当幸福来敲门》。

#### 3. 神经推理与深度学习

这是当前最前沿的方向，旨在让神经网络具备推理能力。

- **可微推理：** 设计特殊的神经网络结构，使其能够模拟逻辑推理步骤。例如，神经图灵机、记忆网络、Transformer的注意力机制本身可以被视为一种在信息间建立联系的软推理。
- **预训练语言模型的隐性推理：**
  - 像GPT、BERT这样的大模型，在海量数据上训练后，似乎隐式地学会了某种程度的推理。
  - 当你向GPT提问：“如果A导演擅长悬疑片，而他的新片是喜剧片，我该期待悬疑感吗？” 它很可能回答“不应该，因为导演更换了类型”。这背后是它从训练数据中学习到的逻辑模式，而不是简单的词语匹配。
- **符号主义与连接主义的结合：**
  - 将神经网络的模式识别能力与符号逻辑系统的可解释性、精确推理能力结合起来。例如，用神经网络从文本中抽取逻辑谓词，再用符号推理引擎进行推导。

------

### 总结

**基于推理的方法**代表了AI系统从“感知”走向“认知”的关键一步。它的核心是：

- **目标：** 实现**深度理解**，而不仅仅是浅层分类。
- **手段：** 利用**结构化知识**和**逻辑规则**进行推导。
- **能力：** 处理**反讽、转折、隐含意义**和需要**多步逻辑链**的复杂问题。

在你设计的电影评论系统中，纯推理方法可能不作为核心分类器，但它可以作为一个强大的**增强模块**。例如，先用一个快速的深度学习模型进行初步分类，再对置信度低的样本（如包含转折词、反讽词的句子）启动一个更复杂、更耗时的推理模块进行二次分析，从而大幅提升系统对复杂语言现象的应对能力。

## 符号主义和连接主义

- **符号主义：** 像是一个**严谨的数学家**。他使用精确的符号和公式（如微积分）来一步步推导和证明定理。过程清晰、可解释，但处理模糊、感性的问题（如欣赏一幅画）时很吃力。
- **连接主义：** 像是一个**儿童的大脑**。他不是通过规则学习，而是通过大量看“猫”和“狗”的图片，自己摸索出区别。他可能说不出为什么，但能准确地指认出来。这个过程像个黑箱，但善于处理感知和模式识别任务。

### 符号主义

符号主义，也称为**逻辑主义**或**规则驱动**的AI，是AI发展早期的主流思想。

#### 核心思想

**认知的本质是符号操作。** 智能行为可以通过对物理符号（如单词、逻辑命题）进行形式化的逻辑运算（如推理、演绎）来再现。它认为大脑类似于一个**数字计算机**，通过操作符号来处理信息。

#### 关键特征

1. **物理符号系统假设：** 这是符号主义的基石，由艾伦·纽厄尔和赫伯特·西蒙提出，认为一个物理符号系统具有必要和充分的手段来实现一般的智能行为。
2. **显式知识表示：** 知识被明确地表示为符号和规则。
   - **例如：** 知识可以用 `如果...那么...` 规则、`谓词逻辑`（如 `是人类(苏格拉底)`、`会死(苏格拉底)`）或 `框架`、`语义网络` 来表示。
3. **逻辑推理：** 智能来自于对符号结构进行基于规则的逻辑操作（如演绎、归纳）。
   - **经典例子：** 三段论
     - 规则：所有人都会死。
     - 事实：苏格拉底是人。
     - 推理：因此，苏格拉底会死。
4. **自上而下：** 通常先由人类专家定义好知识和规则，然后系统据此运行。
5. **可解释性强：** 推理过程是透明的，可以一步步追溯，容易理解和解释。

#### 优点与缺点

| 优点                                                         | 缺点                                                         |
| :----------------------------------------------------------- | :----------------------------------------------------------- |
| **高精确性**：在定义明确的领域（如数学证明、棋类游戏），结果精确。 | **知识获取瓶颈**：依赖人类专家将知识手工编码成规则，费时费力，难以扩展。 |
| **可解释性强**：推理过程清晰，是“白箱”模型。                 | **脆弱性**：无法处理规则之外的情况，缺乏常识和灵活性。       |
| **善于处理序列化、高层次推理**。                             | **难以处理感知、不确定性和模糊问题**：如图像识别、语音理解、处理自然语言的歧义。 |

#### 应用实例

- **专家系统**：如MYCIN（医疗诊断系统），它内部有大量的医学规则。
- **定理证明器**。
- **早期的棋类程序**（如深蓝）。
- **基于规则的自然语言处理**。

------

### 连接主义

连接主义，也称为**神经网络**或**并行分布式处理**，是受生物大脑结构启发的范式。它在近几十年占据主导地位。

#### 核心思想

**智能产生于大量简单处理单元（神经元）之间的相互连接和并行活动。** 知识不是被明确存储的，而是**分布式地**存储在神经元之间的连接权重中。学习就是通过调整这些连接的权重来完成的。

#### 关键特征

1. **受生物大脑启发：** 基本计算单元是“人工神经元”，模拟生物神经元。
2. **分布式表示：** 一个概念不是由一个符号表示，而是由网络中许多神经元的激活模式共同表示。
3. **学习而非编程：** 知识是通过**训练**（如反向传播算法）从数据中自动学习得到的，而不是由人类手动编程输入。
4. **容错性和鲁棒性：** 即使部分神经元损坏，整个网络依然能工作。
5. **自下而上：** 从数据中自动发现特征和模式。
6. **“黑箱”特性：** 虽然我们能分析其结构，但通常很难解释为什么一个特定的输入会产生某个输出。

#### 优点与缺点

| 优点                                                   | 缺点                                                   |
| :----------------------------------------------------- | :----------------------------------------------------- |
| **强大的学习和泛化能力**：能从大量数据中学习复杂模式。 | **“黑箱”问题**：决策过程不透明，难以解释和信任。       |
| **对噪声数据容错性好**。                               | **需要大量数据**：性能严重依赖高质量的大规模数据集。   |
| **善于处理感知、识别、低层次认知任务**。               | **计算资源消耗大**。                                   |
| **无需人工定义特征**：能自动发现数据中的相关特征。     | **缺乏符号推理能力**：难以进行高层次的逻辑推理和规划。 |

#### 应用实例

- **深度学习**：如图像识别、语音识别、机器翻译。
- **你提到的电影评论情感分析系统**（使用BERT、LSTM等模型）。
- **自动驾驶**的视觉感知部分。
- **推荐系统**。

### 融合趋势：神经符号AI

如今，越来越多的研究者认识到，两者并非水火不容，而是**互补**的。真正的通用人工智能可能需要将两者的优势结合起来。这就催生了 **“神经符号AI”**。

**目标：** 将连接主义的**学习能力**和**感知能力**与符号主义的**推理能力**和**可解释性**相结合。

**设想方式：**

1. **感知→符号→推理：** 使用神经网络从原始数据（如图像、文本）中**提取符号概念**（例如，从图片中识别出“猫”、“桌子”），然后将这些符号输入一个符号推理引擎进行逻辑推理。
   - *例子：* 从监控视频中识别出“一个人拿起一个商品”，然后推理出“这是一个顾客”。
2. **符号→神经网络：** 用符号知识来**指导或约束**神经网络的学习过程，使其更高效、更合理。
   - *例子：* 将知识图谱中的规则作为约束，加入到神经网络的训练中，防止它学到违背常识的规律。

**在你项目的语境中：**
一个理想的电影评论系统可能是一个神经符号系统：

- **连接主义部分（神经网络）：** 负责底层文本理解，从评论中提取关键方面，如“演技”、“剧情”、“特效”，并生成初步的情感倾向。
- **符号主义部分（知识库与推理）：** 拥有一个关于电影常识的知识图谱（如“导演A擅长悬疑片”）。当评论中出现“这部电影的导演是A，但悬疑感全无”时，推理引擎能结合知识进行判断，识别出这种“违背预期”的强烈负面情绪，从而修正或增强神经网络的判断。

总而言之，符号主义和连接主义是通往智能之路上两条不同的河流，它们曾经分流，但如今正有汇合的趋势，共同奔向更广阔的人工智能海洋。



# RNN

**1. 传统神经网络的局限**
想象一下，你要用一个标准的前馈神经网络来预测句子中的下一个词。网络会接收一个固定大小的输入（比如几个词），然后输出一个预测。问题是，它**完全忘记了**之前处理过的任何信息。每一次预测都是独立的。这对于序列数据（如语言、时间序列）是致命的，因为上下文至关重要。

**2. RNN的灵感来源**
RNN的灵感来源于人类阅读：我们理解一个词时，会依赖于对前面所有词的理解。我们的大脑会维持一个关于当前所读内容的“思维状态”。

RNN通过在其内部引入**循环**来模拟这一点。它拥有一个“内部状态”或“隐藏状态”，这个状态就像是一个**记忆单元**，包含了序列历史信息的摘要。

------

### **详细工作原理——展开循环**

**1. 核心计算单元与状态传递**

让我们聚焦于RNN中最小的计算单元。在每个时间步 `t`，它执行以下操作：

- **输入：**
  - $ x_t$： 当前时间步的输入向量（例如，一个词的词向量）。
  - $h_{t-1}$： 上一个时间步的**隐藏状态**（即“记忆”）。
- **输出：**
  - $h_t$： 更新后的当前隐藏状态，它将被传递给下一个时间步。
  - $y_t$：（可选）当前时间步的输出。

**2. 数学公式**

RNN的核心计算由以下两个公式定义：

**公式一：更新隐藏状态（记忆）**

pytorch仅使用公式一作为RNN层

**RNN模型的API自定义激活函数，无需像CNN那样手动定义。**(在模型参数里指定，默认“tanh")

$$h_t = \tanh(W_{hh} h_{t-1} + W_{xh} x_t + b_h)$$

- $W_{hh}$： 是一个权重矩阵，负责处理上一个隐藏状态 $h_{t-1}$。它决定了“过去的记忆”对当前状态的影响有多大。
- $W_{xh}$： 是一个权重矩阵，负责处理当前输入$ x_t$。它决定了“当前输入”对当前状态的影响有多大。
- $b_h$： 是隐藏状态的偏置向量。
- `tanh`： 是激活函数（也可以用ReLU等），它引入非线性，并将输出值压缩到(-1, 1)之间，帮助稳定梯度。
- 计算第一个时间步时，$h_0$通常初始化为0向量

**这个公式是RNN的灵魂**。它表示新的记忆 $h_t$ 是**旧的记忆 $h_{t-1}$** 和**新的输入 $ x_t$** 的**组合**。



**公式二：计算输出**

**代码实现的区别**：pytorch的实现并没有这一步，在PyTorch的设计中，`nn.RNN` 是一个**基础RNN层**，它的**输出就是每个时间步的隐藏状态**。

```python
# RNN输出: [batch_size, seq_len, hidden_dim]
rnn_output, hidden = self.rnn(embedded)
```

- `output` 包含了**所有时间步的隐藏状态**。如果有多层（`num_layers>1`）则`output`是最后层的所有时间步的隐藏状态。
- `output` 的最后一个维度是 `hidden_size` (20)
- `hidden` 是**最后一个时间步的隐藏状态**，但对于多层RNN，它包含**所有层的最终隐藏状态**。

**如果有全连接层：**这取决于你的任务

$y_t = W_{hy} h_t + b_y$

- $W_{hy}$： 是一个权重矩阵，将隐藏状态映射到输出空间。
- $b_y$： 是输出的偏置向量。
- $y_t$的形状仍然是hidden_size
- （对于分类任务，$y_t$ 后面通常会接一个Softmax函数来得到概率分布）

**3. 参数共享的关键特性**

请注意，权重矩阵 $(W_{hh}, W_{xh}, W_{hy})$ 和偏置 $(b_h, b_y)$ **在所有时间步之间是共享的**。

- **优点1：** 极大减少了模型需要学习的参数数量，无论序列多长，参数规模不变。
- **优点2：** 赋予了模型强大的泛化能力，使其可以将在一个时间步学到的模式应用到所有时间步。

**4. “展开”视图：理解信息流**

将循环结构按时间维度展开后，RNN看起来像一个深层的网络，但每一层的权重是共享的。

通过这个视图，我们可以清晰地看到信息是如何从左向右流动的。隐藏状态$h_t$就像一个“信息胶囊”，从序列开始一路滚动到结束，沿途不断吸收新的信息。

### **RNN的架构模式**

根据输入和输出的序列结构，RNN有多种应用模式，下图清晰地展示了这一点：

**1. 一对一：** 标准神经网络模式，用于非序列任务（如图像分类）。
**2. 一对多：** 序列生成，输入单个内容，输出序列（如图像描述生成）。
**3. 多对一：** 序列分类，输入序列，输出单个判断（如**情感分析**：输入一条评论，输出正面/负面）。
**4. 多对多（同步）：** 序列标注，输入和输出序列等长（如词性标注：每个输入词对应一个词性标签）。
**5. 多对多（异步）：** 序列转换，输入和输出序列不等长（如**机器翻译**：输入中文句子，输出英文句子）。

#### 2. **双向结构**

基础的 RNN 在每个时间步只输出一个隐藏状态，**该状态仅包含来自上文的信息**，而无法利用当前词之后的下文。

对于一些任务而言，这是一个明显的限制。比如在序列标注任务中，模型需要为每个 token 预测一个标签，如果只能参考前文信息，往往难以做出准确判断。



而使用**双向 RNN**（Bidirectional RNN），模型可以**在每个时间步同时利用前文和后文的信息**，从而获得更全面的上下文表示，有助于提升序列标注等任务的预测效果。

双向RNN同时使用两层 RNN：

正向 RNN：按照时间顺序（从前到后）处理序列；

反向 RNN：按照逆时间顺序（从后到前）处理序列。

双向时间步处理同步进行

每个时间步的输出，是正向和反向隐藏状态的组合（例如拼接或求和）。

- output：[batch_size, seq_len, directions * hidden_size] 最后一层所有的隐藏状态

每一层都是一个双向的结构

最后仍然取的是最后一层的输出（蓝框部分）。

**权重矩阵：**
可以看到除了第一层，后续的高层输入数据的形状都一样，事实也确实如此，第一层的权重矩阵与高层的权重矩阵大小显然不一致（第一层有词嵌入向量参加运算，后续都是隐藏状态的运算）

**计算逻辑**：
每一层的时间步双向计算，互不干扰。但是层间仍是交替并行计算（每一个时间步在所有层计算完毕才计算下一个时间步），每个方向在层间传递时，只使用同方向的隐藏状态，**两个方向在层间完全不交互**，计算过程中不需要拼接计算结果，**最后一层才会拼接两个方向的计算结果**

- output：[batch_size, seq_len, directions * hidden_size] 最后一层所有的隐藏状态
- hidden：[ layers * directions,batch_size, hidden_size] 每一层的最后的隐藏状态（即使batch_first=True）

### **RNN的核心挑战——长程依赖问题**

**1. 问题描述**
理论上，$h_t$包含了所有历史信息 $(x_0, x_1, ..., x_t)$。但在实践中，标准RNN（常称为Vanilla RNN）难以学习长序列中远距离词语之间的依赖关系。

例如，在句子“我出生在法国...（中间有很长一段文字）...所以我讲流利的 __？”中，要预测“法语”，模型需要记住很久之前出现的“法国”这个词。标准RNN很难做到这一点。

**2. 根本原因：梯度消失与梯度爆炸**
这个问题源于反向传播训练算法。梯度需要从最后的损失函数，沿着时间步一路传播回最初的时刻。这个过程被称为**通过时间的反向传播**。

在每一步，梯度都需要乘以权重矩阵 `W_{hh}` 的转置。

- **梯度爆炸：** 如果 `W_{hh}` 的谱范数（主要特征值）大于1，梯度会像滚雪球一样指数级增长，导致数值不稳定和模型无法收敛。
- **梯度消失：** 如果 `W_{hh}` 的谱范数小于1，梯度会像雪崩一样指数级衰减到接近0。这意味着远距离时间步的参数几乎得不到更新，它们对最终输出的影响也微乎其微。**模型变得“健忘”**。

**梯度消失比梯度爆炸更难解决，因为爆炸可以通过梯度裁剪来缓解，而消失则使得学习过程从根本上失效。**

- **dropout**: 当num_layers>1时，每一层之间的隐藏状态$h_t^n$的随机失活
- **bias** ：pytorch中有两个偏置，$b_{ih}$和$b_{hh}$，分别是输入到隐藏层的偏置和隐藏层到隐藏层的偏置。形状都是[hidden_size]，计算时PyTorch自动将 `[hidden_size]` 广播为 `[batch_size, hidden_size]`

#### RNN的参数num_layers的含义

Pytorch中的RNN一般有四个参数input_size，hidden_size， num_layers， batch_first=T/F

`num_layers` 参数指定了 **RNN 的层数**，即堆叠在一起的、独立的 RNN 单元的数量。

- `num_layers = 1`**：这是一个**单层 RNN。输入序列直接连接到输出。
- `num_layers > 1`：这是一个**深度** RNN 或**堆叠式** RNN。第一层 RNN 的输出，作为第二层 RNN 的输入，以此类推。

```python
import torch
import torch.nn as nn

# 参数定义
input_size = 100   # 输入特征维度（如词向量维度）
hidden_size = 32   # 每一层RNN的隐藏状态维度
num_layers = 2     # RNN层数
batch_size = 3
seq_len = 5

# 创建深层RNN
rnn = nn.RNN(input_size=input_size,
             hidden_size=hidden_size,
             num_layers=num_layers,
             batch_first=True)

# 输入数据
input_data = torch.randn(batch_size, seq_len, input_size)
# 初始隐藏状态: [num_layers, batch_size, hidden_size]
h0 = torch.zeros(num_layers, batch_size, hidden_size)

# 前向传播
output, hn = rnn(input_data, h0)

print(f"输入数据维度: {input_data.shape}")
print(f"输出数据维度: {output.shape}")     # 注意：这里仍然是 [3, 5, 32]
print(f"最终隐藏状态维度: {hn.shape}")     # [2, 3, 32]
```

**数据流图示（以 `num_layers=2` 为例）**

**关键点解释：**

1. **输入与输出**：

   - 对于整个模型而言，**最终输出 `output` 的维度仍然是 `[batch_size, seq_len, hidden_size]`**。它始终是**最后一层**在所有时间步的隐藏状态。
   - 在上面的例子中，`output` 来自第二层（Layer 1）。

2. **层间传递**：

   - 第一层（Layer 0）在每个时间步 `t` 接收：

     - 来自外部的输入$ x_t$$（对于 t=0）或上一时间步自己的隐藏状态 $$h_{t-1}^{0}$
     - 它计算出$h_t^{0}$

   - 第二层（Layer 1）在每个时间步 `t` 接收：

     - 第一层在相同时间步 `t` 的输出 $$h_t^{0}$$作为其输入。
     - 以及它自己上一时间步的隐藏状态 $$h_{t-1}^{1}$$
     - 它计算出 $h_t^{1}$，这个结果也就是整个模型在时间步 `t` 的最终输出。

     先计算所有层的第一个时间步，再计算第二个...    （纵向计算后横向计算）

3. **隐藏状态 `hn`**：

   - 最终的隐藏状态 `hn` 的维度是 `[num_layers, batch_size, hidden_size]`。
   - 它包含了**每一层**在**最后一个时间步**的隐藏状态。
   - 在上面的例子中，`hn[0]` 是第一层的最终状态，`hn[1]` 是第二层的最终状态。



### QA 输入数据解释

| 输入 | **input**  | 输入序列，形状为(seq_len, batch_size, input_size)，如果 batch_first=True，则为 (batch_size, seq_len,  input_size) |
| ---- | ---------- | :----------------------------------------------------------: |
| 输入 | **h_0**    | 可选，初始隐藏状态，默认为None，形状为 (num_layers × num_directions,  batch_size, hidden_size) |
| 输出 | **output** | RNN层的输出，包含最后一层每个时间步的隐藏状态，形状为  (seq_len, batch_size,  num_directions × hidden_size )，如果如果 batch_first=True，则为(batch_size,  seq_len, num_directions × hidden_size ) |
| 输出 | **h_n**    | 最后一个时间步的隐藏状态，包含每一层的每个方向，形状为 (num_layers ×  num_directions, batch_size, hidden_size) |



##### 一、数据形态：张量维度

在PyTorch等深度学习框架中，RNN的输入通常是一个**三维张量**，其形状为：

```
(batch_size, sequence_length, input_size)
```

当参数 `batch_first=True` 时，这是最常用的格式。

让我们来拆解这三个维度的含义：

1. **`batch_size`**：
   - **是什么**： 一次输入到模型中的**独立序列样本的数量**。目的是并行处理，提高训练效率。
   - **例子**： 如果你想一次处理32条电影评论，那么 `batch_size` 就是32。
2. **`sequence_length`**：
   - **是什么**： 单个序列的**时间步长度**。即这个序列包含多少个“元素”。
   - **例子**：
     - 对于一条有10个词的评论，`sequence_length` 就是10。
     - 对于一段5天的股票价格，`sequence_length` 就是5。
3. **`input_size`** / `feature_size`：
   - **是什么**： 在每个时间步，输入到RNN单元的**特征的维度**（第一层为embedding_dim，若有多层，则后续层输入维度为hidden_size）。
   - **例子**：
     - 如果使用一个100维的词向量表示一个词，那么 `input_size` 就是100。
     - 如果股票数据在每个时间点包含“开盘价”、“最高价”、“最低价”、“收盘价”4个特征，那么 `input_size` 就是4。

##### 二、数据含义：不同场景下的具体内容

RNN输入数据的**具体内容**完全取决于任务类型。以下是几个典型场景：

**场景一：自然语言处理（如你的电影评论分类）**

这是最经典的应用。输入数据是**文本的数值化表示**。

**处理流程：**

1. **原始输入**： "这部电影很棒"
2. **分词**： `["这", "部", "电影", "很", "棒"]`
3. **构建词汇表**： 为每个词分配一个唯一ID。
   - `{“<pad>": 0, "<unk>": 1, "这": 2, "部": 3, "电影": 4, "很": 5, "棒": 6}`
4. **数值化**： 将句子转换为ID序列 `[2, 3, 4, 5, 6]`。
5. **词嵌入**： **这是最关键的一步！** 将每个词ID通过一个 `nn.Embedding` 层，映射为一个稠密的向量。
   - 例如，`“电影” (ID=4)` -> `[0.1, -0.3, 0.8, ..., 0.02]` (一个100维的向量)

**此时，输入RNN的数据是：**

- `batch_size`： 一个批次中评论的条数（如32）。
- `sequence_length`： 每条评论固定后的词数（如50个词，短的填充，长的截断）。
- `input_size`： 词向量的维度（如100）。

**所以，输入张量的形状是 `(32, 50, 100)`**。RNN会按顺序（第1个词，第2个词...第50个词）处理这条评论。

**场景二：时间序列预测（如股票价格预测）**

输入数据是**历史时间序列的观测值**。

**假设我们要用过去5天的数据预测第6天：**

- 每个时间步的特征可以是：`[开盘价， 最高价， 最低价， 收盘价， 成交量]`

**此时，输入RNN的数据是：**

- `batch_size`： 一个批次中这样的“5天序列”有多少个（如64）。
- `sequence_length`： 历史窗口大小（如5天）。
- `input_size`： 每天的特征数量（如5个）。

**所以，输入张量的形状是 `(64, 5, 5)`**。RNN会按时间顺序（第1天，第2天...第5天）处理这个序列。

**场景三：视频分类（判断视频动作）**

输入数据是**视频帧序列**。

**处理流程：**

1. 将视频分成连续的帧。
2. 每一帧通过一个CNN（如ResNet）提取特征。
3. 将CNN输出的特征向量作为每个时间步的输入。

**此时，输入RNN的数据是：**

- `batch_size`： 一个批次中视频的个数（如16）。
- `sequence_length`： 每个视频抽取的帧数（如30帧）。
- `input_size`： 每一帧图像经过CNN后得到的特征向量的维度（如1024）。

**所以，输入张量的形状是 `(16, 30, 1024)`**。RNN会按顺序（第1帧，第2帧...第30帧）处理这个视频。

### QA Embedding层及参数解释

#### **一、核心思想：从“符号”到“向量”**

**Embedding层的根本作用：将离散的符号（如单词、产品ID）映射到连续的、稠密的向量表示。**

- **离散表示问题**：传统的One-Hot编码是稀疏、高维且无法表达语义关系的（所有词向量两两正交）。
- **Embedding的优势**：
  - **稠密低维**：将高维One-Hot向量压缩到低维稠密向量。
  - **语义学习**：在训练过程中，语义相似的词会逐渐拥有相似的向量表示。

**类比**：

- **One-Hot**：像用身份证号代表人，只表示身份，不表达任何特征。
- **Embedding**：像用一份个人简历（包含学历、技能、爱好等多个维度的分数）来代表人，能体现人的特点和相似度。

------

#### **二、Embedding层的关键参数详解**

以PyTorch的 `torch.nn.Embedding` 为例，我们来看它的参数：

python

```python
embedding_layer = nn.Embedding(
    num_embeddings=10000,  # 词汇表大小
    embedding_dim=300,     # 每个词向量的维度
    padding_idx=0,         # 可选：指定填充符的索引，其向量将被强制设为0
    max_norm=None,         # 可选：最大范数，超过时重新归一化
    norm_type=2.0,         # 可选：归一化时的范数类型（如L2范数）
    scale_grad_by_freq=False, # 可选：是否根据词频缩放梯度
    sparse=False           # 可选：是否使用稀疏梯度更新（用于超大词汇表）
)
```



##### **1. `num_embeddings` (词汇表大小)**

- **含义**：你的词汇表中一共有多少个不同的词（或物品）。
- **如何确定**：通常是你数据集中所有唯一词的数量，再加上一些特殊的标记（如`<pad>`, `<unk>`）。
- **例子**：你的电影评论数据集中有9800个不同的词，加上 `<pad>=0` 和 `<unk>=1`，那么 `num_embeddings=9802`。
- **注意**：这个值定义了Embedding矩阵的**行数**。

##### **2. `embedding_dim` (嵌入维度)**

- **含义**：每个词用多少维的向量来表示。
- **如何确定**：这是一个**超参数**，需要根据任务和数据量进行调整。
  - **较小值（50-100）**：适用于小型数据集或简单任务。
  - **中等值（200-300）**：非常通用，也是预训练词向量（如GloVe）的常用维度。
  - **较大值（500+）**：适用于大型数据集和复杂任务，但需要更多数据和计算资源。
- **例子**：在情感分析任务中，`embedding_dim=256` 是一个不错的起点。
- **注意**：这个值定义了Embedding矩阵的**列数**，以及后续RNN/CNN层的 `input_size`。

##### **3. `padding_idx` (填充索引)**

- **含义**：指定一个索引，这个索引对应的词向量在训练过程中**不会被更新**，并且**始终被设置为0**。
- **为什么需要**：为了将不同长度的句子批量处理，我们需要用“填充符”将短句子补长。这些填充符本身没有语义，不应该参与训练和影响模型。
- **例子**：通常我们将索引 `0` 保留给 `<pad>`。设置 `padding_idx=0` 后，所有ID为0的位置，其向量都是零向量。
- **作用**：
  1. **节省计算**：避免在填充位置进行无意义的计算。
  2. **稳定训练**：防止填充符引入噪声。

##### **4. 其他参数**

- `max_norm`：如果词向量的范数超过此值，会被重新缩放。常用于防止向量变得过大。
- `sparse`：如果设为 `True`，在反向传播时会使用稀疏梯度更新，可以节省内存（尤其是在词汇表极大时）。

------

#### **三、Embedding层的工作流程与代码示例**

##### **步骤1：准备输入数据**

输入给Embedding层的是**词的整数索引**，而不是One-Hot向量。

python

```python
import torch
import torch.nn as nn

# 假设我们的词汇表： {<pad>:0, <unk>:1, "好":2, "电影":3, "烂":4, ...} 共1000个词
vocab_size = 1000
embedding_dim = 300

# 创建Embedding层
embedding = nn.Embedding(num_embeddings=vocab_size, 
                         embedding_dim=embedding_dim, 
                         padding_idx=0)

# 输入数据：2个句子，每个句子5个词（已经转换为索引）
# 注意：第二个句子实际只有3个词，后面用0填充到了长度5
input_indices = torch.LongTensor([
    [2, 3, 1, 4, 1],   # 句子1: "好 电影 <unk> 烂 <unk>"
    [3, 2, 4, 0, 0]    # 句子2: "电影 好 烂 <pad> <pad>"
])

print("输入索引张量形状:", input_indices.shape) # torch.Size([2, 5])
```



##### **步骤2：通过Embedding层获取词向量**

python

```python
# 将索引转换为词向量
output_vectors = embedding(input_indices)

print("输出词向量张量形状:", output_vectors.shape) 
# torch.Size([2, 5, 300])
# 解释：[batch_size=2, sequence_length=5, embedding_dim=300]
```



##### **内部发生了什么？**

Embedding层内部维护着一个可训练的权重矩阵，其形状为 `(num_embeddings, embedding_dim)`。当输入索引 `i` 时，Embedding层就**查找**这个矩阵的第 `i` 行，并将其作为输出。

##### **1. 权重初始化**

- Embedding层的权重默认是随机初始化的。
- 在实践中，我们经常用**预训练词向量**（如GloVe、Word2Vec）来初始化它，这能显著提升模型性能，尤其是在训练数据不多的情况下。

python

```python
# 用预训练向量初始化Embedding层（伪代码）
pretrained_embeddings = load_glove_vectors() # 加载预训练矩阵 [vocab_size, embed_dim]
model.embedding.weight.data.copy_(torch.from_numpy(pretrained_embeddings))

# 通常我们固定预训练向量，不更新它们
# model.embedding.weight.requires_grad = False
```

##### **2. 与One-Hot的关系**

从数学上看，Embedding查找等价于**用一个One-Hot向量乘以Embedding矩阵**：
`one_hot(i) @ embedding_matrix = embedding_matrix[i]`
但PyTorch的实现是高效的直接查找，避免了计算庞大的One-Hot向量。

##### **3. 处理未知词**

- 词汇表中必须包含一个 `<unk>`（未知词）标记。
- 当遇到训练时没见过的词时，就将其映射到 `<unk>` 的索引，使用 `<unk>` 的向量来表示。

##### **4. 训练过程的学习**

- **初始状态**：所有汉字向量随机初始化
- **训练过程中**：通过反向传播不断调整向量值
- **最终效果**：语义相似的汉字在向量空间中位置相近

经过训练后，Embedding权重矩阵会编码汉字的语义信息：

# 中文分词

- 字符级分词：把每一个字作为一个token
- 词级分词：词级分词通常依赖词典、规则或模型来识别词语边界。
- 子词级分词：虽然中文没有英文中的子词结构（如前缀、后缀、词根等），但子词分词算法（如 BPE）仍可直接应用于中文。

### 词级分词

代表工具包括 [**jieba**](https://github.com/fxsjy/jieba)**、**[**HanLP**](https://github.com/hankcs/HanLP)等，这些工具广泛应用于传统 NLP 任务中。

jieba分词器提供了多种分词模式，以适应不同的应用场景。

1. 精确模式（默认）
   试图将句子最精确地切开，适合文本分析。

   ```python
   words_generator = jieba.cut(text)  # 返回一个生成器
   for word in words_generator:
       print(word)
   
   words_list = jieba.lcut(text)  # 返回一个列表
   print(words_list)
   
   ```

   

2. 全模式
   把句子中所有的可以成词的词语都扫描出来，分词效果如下：
   小明毕业于北京大学计算机系----->[小|明|毕业|于|北京|北京大学|大学|计算|计算机|计算机系|算机|系]

   ```python
   words_generator = jieba.cut(text, cut_all=True)  # 返回一个生成器
   for word in words_generator:
       print(word)
   
   words_list = jieba.lcut(text, cut_all=True)  # 返回一个列表
   print(words_list)
   
   ```

   

3. 搜索引擎模式
   在精确模式基础上，对长词进一步切分，适合用于搜索引擎分词，分词效果如下：
   [小明|毕业|于|北京|大学|北京大学|计算|算机|计算机|计算机系]

   ```python
   words_generator = jieba.cut_for_search(text)  # 返回一个生成器
   for word in words_generator:
       print(word)
   
   words_list = jieba.lcut_for_search(text)  # 返回一个列表
   
   ```

4. 自定义词典
   jieba支持用户自定义词典，以便包含 jieba 词库里没有的词，用于增强特定领域词汇的识别能力。
   自定义词典的格式为：一个词占一行，每一行分三部分：词语、词频（可省略，词频决定某个词在分词时的优先级。词频越高被优先切分出来的概率越大）、词性标签（可省略，不影响分词结果），用空格隔开，顺序不可颠倒。例如:

   >云计算
   >
   >云原生 5
   >
   >大模型 10 n

   可使用jieba.load_userdict(file_name)加载词典文件，也可以使用jieba.add_word(word, freq=None, tag=None)与jieba.del_word(word)动态修改词典。

   ```python
   jieba.load_userdict('dict.txt')
   words_list = jieba.lcut("随着云计算技术的普及，越来越多企业开始采用云原生架构来部署服务，并借助大模型能力提升智能化水平，实现业务流程的自动化与智能决策。")
   print(words_list)
   
   ```

## 语义化词向量

词向量的获取通常有两种方式：一种是直接使用他人公开发布的词向量，另一种是在特定语料上自行训练。

在实际工作中，无论是加载已有模型还是从零训练，都可借助[Gensim](https://radimrehurek.com/gensim/)来完成，它提供了便捷的接口来加载 Word2Vec 格式的词向量，也支持基于自有语料训练属于自己的词向量模型

Word2Vec提供了两种典型的模型结构，用于实现对词向量的学习：

- **CBOW（Continuous Bag-of-Words）模型**
  输入是一个词的上下文（即前后若干个词），模型的目标是预测中间的目标词。

-  **Skip-gram** **模型**

# LSTM

为了缓解RNN梯度消失或者梯度爆炸的问题，Hochreiter 和 Schmidhuber 于 1997 年提出了长短期记忆网络（Long Short-Term Memory, LSTM）。

## 结构

LSTM 通过引入特殊的记忆单元（Memory Cell，图中的$C_t$），有效提升了模型对长序列依赖关系的建模能力。其沿时间步展开后的结构如下图所示：

其内部结构如下图所示，核心结构是三个“门”，分别是遗忘门、输入门和输出门。

![image-20251120131637134](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511201316254.png)

- **记忆单元$C_t$：**
  记忆单元负责在序列中长期保存关键信息。它相当于一条“信息通道”，在多个时间步之间直接传递信息（记忆单元是缓解梯度消失和梯度爆炸问题的核心）

- **遗忘门：**
  遗忘门决定当前时间步要忘记多少过去的记忆。
  遗忘门会根据上一个时间步的隐藏状态$h_{t-1}$和当前时间步的输入$ x_t$，生成一个0到1之间的控制系数，然后与上一个时间步的记忆单元状态$C_{t-1}$相乘（哈达玛积），从而动态调整哪些信息应该被遗忘。
  ![image-20251120132137926](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511201321030.png)
  图中的⊙符号为阿达玛乘积（Hadamard product)，表示逐元素相乘。

  计算公式为：
  ![image-20251120132200671](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511201322781.png)

- **输入门：**
  输入门控制要从当前时间步的输入向记忆单元存入多少新的信息。当前时间步的信息由当前输入$ x_t$和上一个隐藏状态$h_{t-1}$计算而成，同时输入门也由当前输入$ x_t$和上一个隐藏状态$h_{t-1}$  计算而成，然后新的信息和输入门相乘得到需要存入记忆单元的信息，如下图所示
  ![image-20251120132505680](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511201325771.png)
  ![image-20251120132545472](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511201325570.png)
- **记忆单元的更新**：
  ![image-20251120132627591](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511201326679.png)
- **输出门：**
  输出门控制从记忆单元中读取多少信息作为当前时间步的隐藏状态进行输出。输出门同样由当前时间步的输入$ x_t$和上一个时间步 $h_{t-1}$ 的隐藏状态计算而成，如下图所示。
  ![image-20251120132756384](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511201327515.png)

### 缓解梯度爆炸和梯度消失

LSTM通过引入记忆单元（Memory Cell），在时间步之间提供了一条稳定的梯度传播路径。

记忆单元的更新公式为：
$$
C_t = f_t\bigodot C_{t-1}+i_t\bigodot\widetilde{h_t}
$$
所以$\frac{\partial C_t}{\partial C_{t-1}}=f_t$

在反向传播时，沿记忆单元路径，梯度传播实际上是多个$ f_t$连乘的结果。虽然每个$ f_t$的取值小于1，但通常较接近于1。这是因为$ f_t$由遗忘门生成，在一般任务中，遗忘门倾向于“记得多、忘得少”，因此$ f_t$的值通常较大。

由于乘积中的每一项$ f_t$较接近1，整体衰减速度远小于传统RNN中隐藏状态链式传播时的指数衰减。这使得早期时间步的输入，能够通过记忆单元路径稳定地影响到最终的总梯度，从而有效参与参数的更新，保证了模型对长序列依赖的学习能力。

# Seq2Seq

传统的自然语言处理任务（如文本分类、序列标注）以静态输出为主，其目标是预测固定类别或标签。然而，现实中的许多应用需要模型动态生成新的序列，如机器翻译，文本摘要，问答对话系统。为了解决这类问题，研究者提出了**Seq2Seq（Sequence to Sequence，序列到序列）模型**。

## 模型结构

Seq2Seq 模型由一个编码器（Encoder）和一个解码器（Decoder）构成。编码器负责提取输入序列的语义信息，并将其压缩为一个固定长度的上下文向量（Context Vector）；解码器则基于该向量，逐步生成目标序列。

![image-20251122230124399](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511222301569.png)

### 1 编码器

编码器主要由一个循环神经网络（RNN/LSTM/GRU）构成，其任务是将输入序列的语义信息提取并压缩为一个上下文向量。

在模型处理输入序列时，循环神经网络会依次接收每个token的输入，并在每个时间步步更新隐藏状态。每个隐藏状态都携带了截止到当前位置为止的信息。随着序列推进，信息不断累积，最终会在最后一个时间步形成一个包含整句信息的隐藏状态。

这个最后的隐藏状态就会作为上下文向量（context vector），传递给解码器，用于指导后续的序列生成。

![image-20251122231028346](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511222310493.png)

### 2 解码器

解码器主要也由一个循环神经网络（RNN / LSTM / GRU）构成，其任务是基于编码器传递的上下文向量，逐步生成目标序列。

![image-20251122231054294](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511222310451.png)

1. 在生成开始时，循环神经网络以上下文向量作为初始隐藏状态，并接收一个特殊的起始标记 <sos>（start of sentence）作为第一个时间步的输入，用于预测第一个 token。

2. 随后，在每一个时间步，模型都会根据前一时刻的隐藏状态和上一步生成的 token，预测当前的输出。这种“将前一步的输出作为下一步输入”的方式被称为自回归生成（Autoregressive Generation），它确保了生成结果的连贯性。

3. 生成过程会持续进行，直到模型生成了一个特殊的结束标记 <eos>（end of sentence），表示句子生成完成。


说明：起始标记和结束标记会在训练数据中显式添加，模型会在训练中学会何时开始、如何续写，以及何时结束，从而掌握完整的生成流程。

> **需要特别注意的是，训练阶段与推理阶段的解码策略是不同的：**
>
> 在推理阶段，解码器采用**自回归生成**方式：每一步的输入是模型自己上一步的预测结果。
>
> 而在训练阶段，通常使用一种称为 **Teacher Forcing** 的策略，即：解码器每一步的输入不是模型上一步的预测结果，而是目标序列中真实的前一个token。

**计算损失**

解码器每一步输出一个token的概率分布，我们通过交叉熵损失函数衡量模型对真实词的预测质量。训练过程中，每一个时间步都会产生一个损失值。该样本的总损失，就是所有时间步的损失值逐步累加的结果。

![image-20260210203502576](C:\Users\aurora\Desktop\研究生\研究生任务\笔记\picture_bed\image-20260210203502576.png)

**反向传播**

在 PyTorch 中，调用 loss.backward() 即可自动完成梯度的反向传播。系统会沿时间维度展开计算图，自动完成所有参数的梯度计算，无需手动推导，实现简洁高效。

# 注意力机制

在seq2seq的基础上，在解码器自回归生成时使用Attention机制，计算注意力分数，让模型学会在生成token时应该更多地关注编码器结果中的哪一个隐藏状态。

- 其他在Seq2Seq基础上不变
- 在解码器中增加注意力评分机制
- 计算注意力结果并融合（拼接）当前解码器的隐藏状态

 **点积评分（Dot）**

点积评分是注意力机制中最简单、最直接的一种相关性评分方法。它通过计算解码器当前时间步的隐藏状态与编码器每个时间步的隐藏状态的点积，来衡量二者之间的相关性：

![image-20260210222944336](C:\Users\aurora\Desktop\研究生\研究生任务\笔记\picture_bed\image-20260210222944336.png)

其含义可以理解为：如果两个向量方向越一致（即越接近），它们的点积就越大，表示相关性越强，模型应当给予更多注意力。

 **通用点积评分（General）**

通用点积评分在点积的基础上引入了一个可学习的权重矩阵W,用于先对编码器隐藏状态进行线性变换，再与解码器隐藏状态进行点积：

![image-20260210224551694](C:\Users\aurora\Desktop\研究生\研究生任务\笔记\picture_bed\image-20260210224551694.png)

该方法的设计动机主要是为了解决编码器和解码器隐藏状态维度不一致的问题。通过引入权重矩阵W，不仅实现了维度对齐，也增强了模型对编码器输出的适应能力，从而提升了注意力机制的表达能力。

**拼接评分（Concat）**

拼接评分是一种表达能力更强的相关性评分方法。它的核心思想是：将解码器当前隐藏状态与编码器每个时间步的隐藏状态拼接为一个长向量，经过线性变换和非线性激活，最后用一个向量进行投影，得到最终打分值：

![image-20260210224659199](C:\Users\aurora\Desktop\研究生\研究生任务\笔记\picture_bed\image-20260210224659199.png)



相比前两种方法，Concat 评分方式在建模能力上更强。它不仅考虑了两个状态的数值关系，还引入非线性变换，能够捕捉更复杂的交互模式，更适合处理对齐关系复杂的任务场景。





# Transformer

Seq2Seq模型通过注意力机制取得了一定提升，但由于整体结构仍依赖 RNN，依然存在计算效率低、难以建模长距离依赖等结构性限制。

Google在2017 年发表一篇论文《[Attention Is All You Need](https://arxiv.org/pdf/1706.03762)》，提出了一种全新的模型架构——Transformer。该==模型完全摒弃了 RNN 结构，转而使用注意力机制直接建模序列中各位置之间的关系==。通过这种方式，Transformer不仅显著提升了训练效率，也增强了模型对长距离依赖的建模能力。

## 模型解释

注意力机制不仅是信息提取的工具，其本质是在每一个目标位置上，显式建模该位置与源序列中各位置之间的依赖关系。

与此同时，循环神经网络（RNN）作为 Seq2Seq 模型的核心结构，其作用也在于建模序列中的依赖关系。通过隐藏状态的递归传递，RNN 使当前位置的表示能够整合前文信息，从而隐式捕捉上下文依赖。==从功能角度看，RNN 与注意力机制完成的是同一类任务：建立序列中不同位置之间的依赖联系。==既然注意力机制也具备建模依赖关系的能力，那么理论上，它就可以在功能上替代 RNN。

此外，==相比 RNN，注意力机制在结构上具备明显优势：无需顺序计算，便于并行处理；任意位置间可直接建立联系，更适合捕捉长距离依赖。==因此，它不仅具备替代的可能，也在效率与效果上表现更优。

### 整体结构

Transformer 的整体结构延续了 Seq2Seq 模型中 “编码器-解码器” 的设计理念，其中，编码器（Encoder）负责对输入序列进行理解和表示，而解码器（Decoder）则根据编码器的输出逐步生成目标序列。

与基于 RNN 的 Seq2Seq 模型一样，Transformer 的解码器采用自回归方式生成目标序列。不同之处在于，每一步的输入是此前已生成的全部词，模型会输出一个与输入长度相同的序列，但我们只取最后一个位置的结果作为当前预测。这个过程不断重复，直到生成结束标记 <eos>。

![image-20251128125500136](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281255263.png)

可以堆叠多层，通过层层堆叠，模型能够逐步提取更深层次的语义特征，从而增强对复杂语言现象的建模能力。标准的 Transformer 模型通常包含 6个编码器层和 6 个解码器层。

![image-20251128130609311](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281306418.png)

### 编码器

每个 Encoder Layer的主要任务都是对其输入序列进行上下文建模，使每个位置的表示都能融合来自整个序列的全局信息。每个 Encoder Layer都包含两个子层（sublayer），分别是自注意力子层（Self-Attention Sublayer）和前馈神经网络子层（Feed-Forward Sublayer）。

![image-20251128134500809](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281345924.png)

- **Self-Attention**：用于捕捉序列中个位置的依赖关系

- **Feed-Forward**：用于对每个位置的表示进行非线性变换，从而提升模型的表达能力

#### 自注意力层

自注意力机制（Self-Attention）是 Transformer 编码器的核心结构之一，它的作用是在序列内部建立各位置之间的依赖关系，使模型能够为每个位置生成融合全局信息的表示。

##### 1 生成Query、Key、Value向量。

![image-20251128134828267](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281348368.png)

- Query：表示当前词的用于发起注意力匹配的向量；

- Key：表示序列中每个位置的内容标识，用于与 Query 进行匹配；

- Value：表示该位置携带的信息，用于加权汇总得到新的表示。

**自注意力的核心思想是：每个位置用自身的 Query 向量，与整个序列中所有位置的 Key 向量进行相关性计算，从而得到注意力权重，并据此对对应的 Value 向量加权汇总，形成新的表示。**

![image-20251128134937839](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281349922.png)

其中$W_q,W_k,W_v$均为可学习的参数矩阵。

**划分Q、K、V的原因是**：三者分别学习各自的职责规律，而不是用一个向量承担多种规律的学习（相似度计算规律，承载自身信息职责等），导致学习效果不好

##### 2 **计算位置间相关性**

![image-20251128135113595](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281351700.png)

评分函数采用向量点积形式。由于在高维空间中，点积的数值可能过大，会影响 softmax 的稳定性，因此在实际计算中对结果进行了缩放。最终的评分函数为：
$$
score(i,j)=\frac{q_i\bullet k_j}{\sqrt{d_k}}
$$
其中$d_k$是key向量的维度，用于缩放点积的幅度。这个分数越大，表示第 i 个位置越应该关注第 j 个位置的信息。

**原因**：假设查询 `Q` 和键 `K` 的每个分量都是均值为0、方差为1的独立随机变量。那么，点积 `Q · K` 的均值为0，而方差为 `d_k`。

**推导如下：**

- 方差的性质：`Var(XY) = E[X²]E[Y²] - (E[X]E[Y])²`（当X和Y独立时）
- 由于均值为0，方差为1，所以 `E[X²] = E[Y²] = 1`。
- 因此，`Var(XY) = 1 * 1 - 0 = 1`。
- 点积 `Q · K` 是 `d_k` 个这样的独立乘积之和：`Var(Q·K) = d_k * Var(XY) = d_k`。

**结论：点积 `Q · K` 的值的幅度（标准差）会随着 `sqrt(d_k)` 增长。** `d_k` 越大，点积的结果就越大（绝对值上），也就更容易出现极大值。那么softmax函数会将值映射为0或1，最终导致梯度消失。所以要进行缩放：`(Q · K^T) / sqrt(d_k)` 的方差就变为了 `d_k / (sqrt(d_k))^2 = 1`。

##### 3 计算注意力权重

在得到每个位置与所有位置之间的相关性评分后，模型会使用 softmax 函数进行归一化，确保每个位置对所有位置的关注程度之和为 1，从而形成一个有效的加权分布。

![image-20251128162735499](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281627595.png)

##### 4 加权汇总生成输出

最后，模型会根据注意力权重对所有位置的 Value 向量进行加权求和，得到每个位置融合全局信息后的新表示。

![image-20251128162852178](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281628278.png)



综上所述，完整的流程如下图示
$$
Attention(Q,K,V)=softmax(\frac{QK^T}{\sqrt{d_k}})V
$$
![image-20251128162956093](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281629216.png)

#### 前馈神经网络层

前馈神经网络（Feed-Forward Network，简称 FFN）是 Transformer 编码器中每个子层的重要组成部分，紧接在多头注意力子层之后。它通过对每个位置的表示进行**逐位置**、**非线性**的特征变换，进一步提升模型对复杂语义的建模能力。（**主要是加一个非线性变换**）
$$
FFN(x)={Linear}_2(ReLU({Linear}_1(x)))=W_2\bullet ReLU(W_1x+b_1)+b_2
$$
![image-20251128163738184](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281637273.png)

#### 残差链接与层归一化

在 Transformer 的每个编码器层中，每个子层，包括自注意力子层和前馈神经网络子层，其输出都要经过**残差连接（Residual Connection）和层归一化（Layer Normalization）**处理。这两者是深层神经网络中常用的结构，用于**缓解模型训练中的梯度消失、收敛困难等问题**，对于Transformer能够堆叠多层至关重要。

![image-20251128163843096](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281638195.png)

1. **残差链接**：借鉴ResNet思想，最初在计算机视觉领域被提出，用于缓解深层神经网络中的梯度消失问题。
   $$
   y=x+SubLayer(x)
   $$
   如何缓解梯度消失：y的梯度至少为1，不会小于1导致梯度消失
   
2. **层归一化**：
   每个子层在残差连接之后都会进行**层归一化（Layer Normalization，简称 LayerNorm）**。它的主要作用是规范输入序列中每个token的特征分布（某个token的表示可能在不同维度上有较大数值差异），提升模型训练的稳定性。
   该操作会将每个token的向量调整为均值为 0、方差为 1 的规范分布。
   $$
   LayerNorm\left(x^i\right)=\gamma^i\cdot\widehat{x^i}+\beta^i
   $$
   $\gamma^i和\beta^i$为可学习参数。让模型可以学习在归一化后的基础上进行适当的调整，保证归一化不会限制模型的表示能力（否则可能强制模型服从某一规律）。
   
   如何加速收敛：缓解**内部协变量偏移**（自行搜索）。**平滑了损失函数的 landscape（优化地形），使梯度更稳定。**

#### 位置编码

Transformer 模型完全摒弃了 RNN 结构，意味着它不再按顺序处理序列，而是可以并行处理所有位置的信息。尽管这带来了显著的计算效率提升，但Transformer 无法像 RNN 那样天然地捕捉词语之间的顺序关系。

为了解决这一问题，Transformer 引入了一个关键机制——位置编码（Positional Encoding）。该机制为每个词引入一个表示其位置信息的向量，并将其与对应的词向量相加，作为模型输入的一部分。

![image-20251128173225806](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281732913.png)

Transformer 使用了一种基于正弦（sin）和余弦（cos）函数的位置编码方式，具体定义如下：
$$
{PE}_{\left(pos,2i\right)}=\sin(\frac{pos}{{10000}^\frac{2i}{d_{model}}})
$$

$$
{PE}_{\left(pos,2i+1\right)}=\cos(\frac{pos}{{10000}^\frac{2i}{d_{model}}})
$$

- *pos*是当前词在序列中的位置；

- *i*用于表示位置编码向量的维度索引，*2i*表示偶数维，*2i+*1表示奇数维；

- $d_{model}$是词向量的维度大小。



![image-20251128173605701](C:\Users\aurora\AppData\Roaming\Typora\typora-user-images\image-20251128173605701.png)

Transformer提出的这种编码方式不依赖任何可学习参数，数值稳定，并具备以下优势：

- 所有值都在[−1,1]范围内，数值稳定
- 编码方式固定、可预计算，无需训练
- 相同位置的编码在不同句子中保持一致
- 编码之间具有数学规律，便于模型在注意力机制中感知词语之间的相对位置关系



**位置编码可以感知到token之间的相对位置**：在计算自注意力的时候有一部分信息只与相对位置的距离相关，所有可以学习到相对位置信息。





### 解码器

Transformer 解码器的主要功能是：根据编码器的输出，逐步生成目标序列中的每一个词。其生成方式采用**自回归机制（autoregressive）**：每一步的输入由此前已生成的所有词组成，模型将输出一个与当前输入长度相同的序列表示。我们只取最后一个位置的输出，作为当前步的预测结果。这一过程会不断重复，直到生成特殊的结束标记 <eos>，表示序列生成完成。

![image-20251128174105711](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281741835.png)

每个Decoder Layer都包含三个子层，分别是**Masked自注意力子层**、**编码器-解码器注意力子层（Encoder-Decoder Attention）**和**前馈神经网络子层（Feed-Forward Network）**。

- **Masked自注意力子层**：
  用于建模当前位置与前文词之间的依赖关系。为了在训练时模拟逐词生成的过程，引入遮盖机制（Mask），限制每个位置只能关注它前面的词。
- **编码器-解码器注意力子层**：
  用于建模当前解码位置与源序列各位置之间的依赖关系。通过注意力机制，模型能够根据当前状态从编码器的输出中提取相关上下文信息（相当于 Seq2Seq 模型中的 Attention 机制）。
- **前馈神经网络子层**：
  与编码器中结构完全一致，对每个位置的表示进行非线性变换，增强模型的表达能力。

每个子层后也都配有**残差连接**与**层归一化（Layer Normalization）**，结构设计与编码器保持一致，确保训练的稳定性和效率。

![image-20251128174907117](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281749214.png)

为了确保训练稳定，每个子层之后都配有**残差连接与层归一化（LayerNorm）**，与编码器的设计保持一致，便于模型堆叠和优化。

此外，解码器在输入端同样需要加入**位置编码（Positional Encoding）**，用于提供序列中的位置信息，其计算方式与编码器中相同。

在输出端，解码器的隐藏向量会送入一个**线性变换层（Linear）**，映射为词表大小的向量，并通过 **Softmax** 生成一个概率分布，用于预测当前应输出的词。

#### 1 Masked 自注意力子层

由于 Transformer 不具备像 RNN 那样的隐藏状态传递机制，无法在序列生成过程中保留上下文信息，因此在生成每一个词时，必须将此前已生成的所有词作为输入，通过自注意力机制重新建模上下文关系，以预测下一个词。

从结构上看，Transformer 编解码器都具有一个典型特性：**输入多少个词，就输出多少个表示**。需要注意的是，在推理阶段，我们只使用解码器**最后一个位置的输出**作为当前步的预测结果(推理阶段)，如下图所示：

![image-20251128175140792](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281751883.png)

训练阶段：需要发挥Transformer并行计算的特点，但又不能让当前时间步看到后面的词，破坏生成任务的因果结构。由此需要引入**掩盖机制（Mask），**该机制会在计算注意力时，**阻止模型访问当前位置之后的词**，只允许它依赖自身及前文的信息。这样，即使在并行训练时，模型也只能像逐词生成一样“看见”它应该看到的内容，从而保持训练与推理阶段的一致性。如下图所示：

![image-20251128175530173](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281755260.png)

Mask 机制的实现非常简单：只需将注意力得分矩阵中**当前位置对其后续位置的评分设置为** **−∞**，如下图所示：

![image-20251128175551122](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281755209.png)

这样，在经过 softmax 运算后，这些位置的权重会趋近于 0。最终在加权求和时，来自未来位置的信息几乎不会参与计算，从而实现了“**当前词只能看到它前面的词**”的约束。如下图所示：

![image-20251128175718970](https://raw.githubusercontent.com/Aurora234/picture_bed/main/QA_pic/202511281757061.png)

#### 2 编码器-解码器注意力子层

编码器-解码器注意力的核心机制与前面讲过的自注意力机制完全一致，区别仅在于：

Query 来自解码器当前的输入表示，即当前生成状态；

Key和Value 来自编码器的输出表示，即整个源序列的上下文。

也就是说，当前生成位置使用自己的Query，去“询问”编码器输出中的哪些位置最相关。注意力机制会根据 Query 与所有 Key 的相似度，为每个源位置分配一个权重，然后用这些权重对 Value 进行加权求和，得到当前生成词所需的上下文信息。

### API

PyTorch 提供了对 [Transformer ](https://docs.pytorch.org/docs/stable/nn.html#transformer-layers)的官方实现，该模块封装了完整的编码器-解码器结构，可直接应用于机器翻译、文本生成等典型的序列建模任务。

PyTorch 中的 Transformer 模块由以下几个核心类构成：

- **nn.Transformer**
  封装了完整的 Transformer架构，由编码器和解码器组成。作为顶层接口，适用于需要同时使用编码器和解码器的任务，如机器翻译。支持用户通过参数自定义层数、注意力头数、隐藏维度等模型结构。
- **nn.TransformerEncoder**
  实现了Transformer编码器结构，由多个编码器层的堆叠而成，用于将输入序列编码为上下文相关的表示。
- **nn.TransformerDecoder**
  实现了Transformer解码器结构，由多个解码器层堆叠而成，用于基于编码结果逐步生成目标序列。
- **nn.TransformerEncoderLayer**
  实现了单个编码器层结构，包含一个多头自注意力子层和一个前馈神经网络子层，两者均带有残差连接和 LayerNorm。
- **nn.TransformerDecoderLayer**
  实现了单个解码器层结构，包含自注意力、编码器-解码器注意力、前馈子层，同样配有残差连接和 LayerNorm。

#### Transformer构造参数

```python
torch.nn.Transformer(d_model=512, 
                     nhead=8, 
                     num_encoder_layers=6, 
                     num_decoder_layers=6, 
                     dim_feedforward=2048, 
                     dropout=0.1, 
                     activation='relu', 
                     custom_encoder=None, 
                     custom_decoder=None, 
                     layer_norm_eps=1e-05, 
                     batch_first=False, 
                     norm_first=False, 
                     bias=True, 
                     device=None, 
                     dtype=None)
# 示例，通常只使用几个重要参数
# 初始化 Transformer
transformer = nn.Transformer(
    d_model=512, 
    nhead=8, 
    num_encoder_layers=6, 
    num_decoder_layers=6, 
    batch_first=True
)
```

| **参数名**             | **默认值** | **说明**                                                     |
| ---------------------- | ---------- | ------------------------------------------------------------ |
| **d_model**            | 512        | 编码器和解码器输入的特征维度，即每个 token 的表示向量长度。  |
| **nhead**              | 8          | 多头注意力中的头数，要求  d_model 能被整除。                 |
| **num_encoder_layers** | 6          | 编码器中堆叠的层数，决定模型深度。                           |
| **num_decoder_layers** | 6          | 解码器中堆叠的层数，结构与编码器相似。                       |
| **dim_feedforward**    | 2048       | 前馈神经网络中的隐藏层维度，默认是 d_model 的 4 倍。         |
| **dropout**            | 0.1        | Dropout 概率，用于缓解过拟合，作用于注意力层和前馈层。       |

#### 掩码注解

##### **src_key_padding_mask**

编码器pad掩码， 形状`(batch_size, src_len)`，在计算子注意力分数时应用，将形状扩展为`[batch_size, num_heads, src_len, src_len]`，如下代码扩展:

```python
mask_self = src_key_padding_mask.unsqueeze(1).unsqueeze(2)  # [batch_size,1, 1, src_len]
mask_self = mask_self.expand(batch_size, num_heads, src_len, src_len)
```

如此，可以将对pad的注意力的值变为负无穷。即注意力分数矩阵后面几列的值为负无穷，但后面几行是pad对应的行，pad仍然会尝试理解有效token。

##### tgt_key_padding_mask

解码器pad掩码，形状` (batch_size, tgt_len)`，它会在掩码自注意力子层和交叉注意力子层作用。

1. 掩码自注意力子层：作用与`src_key_padding_mask`一致，扩展后形状为`[batch_size, num_heads, tgt_len, tgt_len]`

2. 交叉注意力子层：作用在Query，扩展方式与在掩码自注意力子层不同：
   ```python
   mask_self = src_key_padding_mask.unsqueeze(1).unsqueeze(3)  # [batch_size,1, tgt_len, 1]
   mask_self = mask_self.expand(batch_size, num_heads, tgt_len, src_len)  #[batch_size,num_heads, tgt_len, src_len]
   ```

   它会将解码器输入的Query后面的pad对应的行掩盖（变为负无穷）

##### tgt_mask

解码器掩码自注意力层因果掩码，作用于自注意力层的注意力分数矩阵，上三角矩阵，形状 `(tgt_len, tgt_len) `，让当前的token只关注前面的token，使解码器也可以并行进行计算，而不是完全的自回归生成

##### memory_key_padding_mask

 解码器交叉注意力子层pad掩码，作用于注意力分数计算，作用形式与`src_key_padding_mask`一致。

它的作用与`src_key_padding_mask`一样，但它存在的意义是作为“二次保险“，原因如下：

1. **编码器没有完美处理<pad>**
   即使编码器使用了 `src_key_padding_mask`，`<pad>` 位置的输出**可能不是完美的零向量**：

   - 由于残差连接、层归一化等操作，`<pad>` 位置的输出可能包含一些残留信息
   - 在深层网络中，信息会通过多种路径传播
   - 某些架构或实现中，编码器输出中的 `<pad>` 位置可能仍有非零值

   在这种情况下，`memory_key_padding_mask` 提供了额外的保护。

2. **数值稳定性**
   即使 `<pad>` 位置的输出值很小，但在 Softmax 中仍然可能产生非零的注意力权重：

3. **训练效率与稳定性**
   - **明确的指令**：`memory_key_padding_mask` 给模型一个明确的信号——"完全忽略这些位置"
   - **减少噪声**：消除 `<pad>` 位置的微小影响，让模型专注于学习有效模式
   - **训练稳定性**：避免模型在噪声中寻找不存在的模式

| ------------------------ | ------------------------------------------------------------ |

#### Transformer.decoder

Transformer模型中的解码器部分，可通过属性 transformer.decoder 访问，其本质是一个 nn.TransformerDecoder 实例。通过其 forward 方法，可以基于编码器的输出（memory）和目标序列的嵌入表示


| --------------------------- | ------------------------------------------------------------ |
| **memory**                  | 编码器的输出表示，包含源序列每个 token 的上下文语义信息，作为解码器的输入。其形状为 (batch_size,  src_len, d_model)。 |

| 输出： **output**           | 解码器输出的隐藏状态序列，形状为 (batch_size, tgt_len, d_model)。表示目标序列中每个位置的上下文表示，通常会送入线性层和 softmax，用于生成词表上的预测概率。 |

**tgt_key_padding_mask**  可以不传，**tgt_mask**  部分代替了它的工作





# 预训练模型

早期的自然语言处理方法通常针对每个具体任务单独训练模型，且严重依赖大量人工标注数据。虽然在部分场景下效果可观，但也暴露出显著局限：

- 语言知识难以复用：每个模型都需从零开始训练，导致训练成本高、效率低；

- 强依赖高质量标注：在医疗、法律等专业领域，标注数据获取困难且代价高昂。

为解决这些问题，研究者提出了新的建模范式——“预训练 + 微调”：

- 预训练阶段：在大规模未标注语料上训练语言模型，学习词汇、句法和上下文等通用语言规律；

- 微调阶段：将预训练模型迁移至具体任务，仅需少量标注数据即可完成任务适配。

## 预训练模型分类

Transformer具有以下优势：

- 并行计算效率高，适合大规模训练；

- 上下文建模能力强，可捕捉长距离依赖；

- 结构通用灵活，可适配多种任务类型；

- 易于扩展与迁移，支持参数堆叠与多任务学习。

分类：

1. 解码器（Decoder-only）模型
   仅使用Transformer解码器，代表模型为**GPT**（Generative Pre-trained Transformer），其由 OpenAI于2018年6月提出，论文题为[《Improving Language Understanding by Generative Pre-Training》](https://cdn.openai.com/research-covers/language-unsupervised/language_understanding_paper.pdf)。



2. 编码器（Encoder-only）模型
   仅使用Transformer 编码器，代表模型为**BERT**（Bidirectional Encoder Representations from Transformers），由Google于2018年10月提出，论文题为[《BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding》](https://arxiv.org/pdf/1810.04805)。



3. 编码器-解码器（Encoder-Decoder）模型
   同时使用Transformer编码器和解码器，代表模型为**T5**（Text-to-Text Transfer Transformer），由Google于2019年10月提出，论文题为《[Exploring the Limits of Transfer Learning with a Unified Text-to-Text Transformer》](https://arxiv.org/pdf/1910.10683)。

### 解码器模型-GPT

GPT（Generative Pre-trained Transformer）是第一个系统性提出“预训练 + 微调”范式的语言模型。

其核心思想是通过大规模无监督语料进行**生成式语言建模**预训练，即训练模型根据左侧上下文预测下一个词，从而让模型学习自然语言的通用语法、语义和上下文依赖能力。完成预训练后，再通过微调适应具体的下游任务。

GPT首次展示了生成式语言模型在自然语言理解任务中的广泛迁移能力，为后续 GPT 系列及整个预训练语言模型的发展奠定了基础。

![image-20260218220314345](C:\Users\aurora\Desktop\研究生\研究生任务\笔记\picture_bed\image-20260218220314345.png)

****

1）输入嵌入层（Text & Position Embedding）

与原始Transformer一样，每个输入 token 的表示也由两部分组成：

- Text Embedding：将词或子词映射为向量；
- Position Embedding：提供词在序列中的位置信息。

GPT不同于原始Transformer的一点在于：位置编码采用的是可学习的位置嵌入（learnable positional embedding）。这意味着每个位置对应一个可训练的向量，模型可以在训练过程中自动优化这些向量，而非使用不可训练的三角函数编码（如正弦/余弦函数）。

每个token的最终表示是词嵌入与位置嵌入的向量和，向量维度为 768。

2）解码器

解码器部分由12个结构相同的解码器层堆叠而成，每个解码器层只包含如下两个子层：

- **掩码多头自注意力（12头）**
- **前馈网络**

3）输出层

根据任务不同，GPT模型的输出可以接入不同的任务头：

**Text Prediction（文本预测）**

用于下一个词的生成，输出是词表大小的概率分布，经过Softmax获得，预训练阶段使用的便是该任务头。

**Task Classifier（任务分类器）**

该任务头多用于模型微调阶段，以适配具体的下游任务。通过提取特定位置的表示（如最后一个token）对整个输入文本进行分类（如情感分析、话题识别等）。

#### 预训练

GPT 的预训练阶段采用生成式语言建模（Generative Language Modeling）作为训练目标，在大规模无监督文本上进行自监督学习。具体而言，模型的任务是基于已观察到的前文上下文，预测当前词的位置应出现的词，从而学习自然语言的统计规律与上下文依赖关系。这种自回归语言建模方式不依赖人工标注，训练样本可以直接从原始文本中自动构建，极大地降低了构建数据的成本。

##### GPT 的预训练流程详解

✅ 1. **模型架构：Transformer 解码器**（Decoder-only）

- 仅使用 **Transformer 的解码器部分**（但去掉 encoder-decoder attention 和 masking for future tokens 已内置）
- 每一层包含：
  - **Masked Multi-Head Self-Attention**（关键！防止看到未来）
  - **Position-wise Feed-Forward Network**
- 使用 **残差连接 + LayerNorm**（Post-LN 结构）

> ⚠️ 注意：GPT 的“解码器”在预训练阶段**没有编码器输入**，只是一个纯自回归生成模型。

---

✅ 2. **输入表示**

- **Token Embedding**：将词映射为向量（使用 BPE 分词）

- **Positional Embedding**：加入绝对位置编码（与原始 Transformer 相同）

- 最终输入：  
  $$
  h_0 = W_e x + W_p
  $$
  其中 $ x $ 是 token ID 序列，$ W_e $ 是词嵌入矩阵，$ W_p $ 是位置嵌入。

---

✅ 3. **训练目标：预测下一个词**

- 对每个位置 $ t $，模型输出一个词汇表大小的 logits 向量
- 使用 **交叉熵损失**（Cross-Entropy Loss）计算预测与真实下一个词的误差
- **所有位置都参与损失计算**（不像 BERT 只算 masked 位置）

```python
# 伪代码
logits = model(input_ids)          # [batch, seq_len, vocab_size]
loss = cross_entropy(
    logits[:, :-1, :].reshape(-1, vocab_size),   # 预测 x2~xT
    input_ids[:, 1:].reshape(-1)                 # 真实标签 x2~xT
)
```

---

✅ 4. **关键机制：因果掩码**（Causal Mask / Look-ahead Mask）

为了保证模型**只能看到当前及之前的词**，在 self-attention 中使用**上三角掩码**：

| Query \ Key | x₁   | x₂   | x₃   | x₄   |
| ----------- | ---- | ---- | ---- | ---- |
| x₁          | ✓    | ✗    | ✗    | ✗    |
| x₂          | ✓    | ✓    | ✗    | ✗    |
| x₃          | ✓    | ✓    | ✓    | ✗    |
| x₄          | ✓    | ✓    | ✓    | ✓    |

- 掩码将未来位置的 attention score 设为 `-inf`
- softmax 后，未来位置权重为 0

> 这就是 `torch.nn.MultiheadAttention` 中 `attn_mask` 或 `is_causal=True` 的作用。



#### 微调

GPT的微调阶段是在完成无监督语言建模预训练之后，使用有监督的任务数据对模型进行进一步训练，使其适应具体的下游任务。微调的核心思路是：在保留预训练语言建模能力的基础上，利用标注数据对整个模型进行端到端优化，从而实现知识迁移。

具体做法如下：

**添加任务输出层**

**统一输入格式设计**

### 编码器模型-Bert

核心创新在于采用 Transformer 的**编码器（Encoder）**结构，通过双向自注意力机制，在建模每个 token 表示时同时整合左右两个方向的上下文信息，从而获得更准确、更丰富的语义表示。

BERT 的设计更侧重于自然语言理解类任务，广泛应用于文本分类、序列标注、句子匹配等场景。模型发布后，在多个语言理解基准测试中取得了前所未有的领先成绩，推动 NLP 研究全面转向“预训练 + 微调”的通用建模范式。

#### 模型结构

BERT 基于**标准的** **Transformer** **编码器**构建，其提供了两种模型规模，分别是BERT-base和BERT-large。

| **模型版本**   | **层数**  **（Layers）** | **模型维度**  **（d_model）** | **注意力头数**  **（Heads）** | **参数量** |
| -------------- | ------------------------ | ----------------------------- | ----------------------------- | ---------- |
| **BERT-base**  | 12                       | 768                           | 12                            | 1.1 亿     |
| **BERT-large** | 24                       | 1024                          | 16                            | 3.4 亿     |

**1）输入表示层**

BERT 的每个输入 token 表示由三部分嵌入相加组成：

Ø **Token Embedding**：词本身的语义表示；

Ø **Position Embedding**：表示 token 在序列中的位置，为可学习向量；

Ø **Segment Embedding**：用于区分句子对任务中的两个句子，分别用一个可学习的向量表示。

**2）编码器**

编码器结构同原始Transformer相同，不再赘述。

**3）输出层**

根据下游任务的类型，BERT 可以接入不同的任务输出头：

Token-Level 任务（如命名实体识别）：使用每个位置的输出表示；

Sequence-Level 任务（如文本分类、句子对分类）：使用特殊 token [CLS] 的输出表示，输入时被加在序列开头，专门用于汇总整个序列的语义信息。

#### 预训练

BERT 的预训练阶段包含两个核心任务：**掩码语言模型（Masked Language Modeling, MLM）** 和 **下一句预测（Next Sentence Prediction, NSP）**，分别用于学习词级语义和句间逻辑关系。

Ø **掩码语言模型（MLM）**

为实现双向语言建模，BERT 不采用传统的从左到右或从右到左预测方式，而是引入了掩码语言模型。在训练中，BERT 会随机遮盖输入序列中约 15% 的 token，并训练模型根据上下文预测被遮盖的词。

遮盖策略如下：

- l  80% 的被遮盖 token 替换为 [MASK]；
- l  10% 替换为随机词；
- l  10% 保持原词不变。

这种机制让模型在预训练时既能看到左侧上下文，也能看到右侧上下文，真正实现深度双向建模。

![image-20260225183259479](C:\Users\aurora\Desktop\研究生\研究生任务\笔记\picture_bed\image-20260225183259479.png)

Ø **下一句预测（NSP****）**

为了提升模型理解句间关系的能力，BERT 引入了“下一句预测”任务。训练时模型接收两个句子，判断第二句是否是第一句的真实后续句，其中：

50% 的训练样本是上下文中真实相邻的句子（正例）；

50% 是从语料中随机采样的非相邻句子（反例）。

Ø **正例：**

- l  A：我今天很忙。
- l  B：所以没去上班。

Ø **反例：**

- l  A：我今天很忙。
- l  B：天气很好。

### 解码器编码器模型-T5

注：该模型不值得学习，了解即可，主要重点学习解码器模型

T5的核心思想是将所有自然语言处理任务统一表示为“文本到文本”的转换问题（Text-to-Text Framework），即无论输入是文本分类、问答还是翻译，模型的输入输出均是自然语言形式的字符串，如下图所示：

![image-20260225184058422](C:\Users\aurora\Desktop\研究生\研究生任务\笔记\picture_bed\image-20260225184058422.png)

这一设计使得 T5 可以通过同一个模型架构、同一套预训练机制完成多种任务，具备极强的统一性与迁移能力。

#### 模型机构

T5模型大体遵循原始的Transformer架构，此处不再赘述。

#### 预训练

T5模型的预训练目标被称为Corrupted span prediction，具体过程如下：

1）随机遮盖输入文本中的若干连续片段（span）；

2）将每个被遮盖的连续片段替换为一个个特殊token；

3）令模型学习生成这些遮盖片段的内容，作为输出序列。

如下图所示：

![image-20260225184814614](C:\Users\aurora\Desktop\研究生\研究生任务\笔记\picture_bed\image-20260225184814614.png)

这种方式既保留了模型的双向建模能力，又为训练提供了明确的“生成式”学习信号，使模型可以更自然的适配下游任务。

#### 微调

T5微调阶段需要将所有任务转换为文本到文本的形式，例如：

| **任务类型** | **输入形式**                                                 | **目标输出** |
| ------------ | ------------------------------------------------------------ | ------------ |
| **翻译**     | translate English to German: That is good.                   | Das ist gut. |
| **情感分类** | sentiment: This movie was great.                             | positive     |
| **问答**     | question: What is the capital of France? context: France is a country... | Paris        |
